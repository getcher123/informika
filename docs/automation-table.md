# Автоматизация (триггеры и действия)

| Триггер | Автоматическое действие | Получатели / Примечание |
|---------|-------------------------|-------------------------|
| Участник отправил идею на модерацию | Уведомление модератору и стейкхолдеру; на административной доске создаётся задача «Проверить новую идею и подготовить решение по публикации» (исполнитель — модератор) | Push/e-mail, ссылка на карточку идеи; задача в колонке «Новая» |
| Модератор вернул идею на доработку | Уведомление участнику с перечнем замечаний | Статус меняется на «На доработке» |
| Модератор изменил статус идеи на «На рассмотрении» | Уведомление стейкхолдеру, идея становится доступна для принятия решения | Фиксируется комментарий/причина |
| Стейкхолдер принял решение («Одобрена» / «Отклонена») | Уведомление участнику (push/e-mail) | Журнал идеи пополняется событием |
| Стейкхолдер отправил идею на доработку | Уведомление автору, запись причины в журнале, возврат статуса «На доработке»; на административной доске создаётся задача «Контролировать доработку автором и повторную подачу» (исполнитель — стейкхолдер) | Причина обязательна; задача в колонке «В работе», ссылка на карточку идеи |
| Дедлайн запроса истёк | Запрос автоматически переводится в «Закрыт», кнопка «Подать идею» скрывается | Все подписчики запроса получают уведомление |
| Идея переведена в «Одобрена» | Создаётся рабочая область в Bitrix24: канбан-доска «<ID идеи>» с колонками `Новая`, `В работе`, `На проверке`, `Возврат`, `Готово`; загружаются шаблонные карточки задач («Аналитика», «План», «Прототип», «Тестирование», «Комплект документов», «Итоговый отчёт»); на Диске формируется структура папок `/Требования`, `/План`, `/Прототипы`, `/Тесты`, `/Отчёты`, `/Коммуникации`; в корень и папку `/План` добавляются гайды `README-по-команде.md`, `task-guide.txt`, шаблон чек-листа и образцы документов; на административной доске создаётся задача «Настроить рабочую область Bitrix24 и запустить приглашения команды» (исполнитель — координатор) | Координатор получает чек-лист подключения, участники — приглашения в Bitrix24, стейкхолдер — ссылку на доску; задача в колонке «Новая» |
| Освобождена обязательная роль в команде | Система ставит предупреждение, требует назначение нового участника; на административной доске создаётся задача «Подобрать кандидата и закрыть обязательную роль» (исполнитель — координатор) | Координатор получает уведомление, событие фиксируется в журнале; задача в колонке «Новая» |
| Участник подал заявку в команду | Заявка появляется у автора/координатора во «Входящих» + уведомление; на административной доске создаётся задача «Рассмотреть заявку участника и принять/отклонить» (исполнитель — координатор) | Статус заявки — «На рассмотрении»; задача в колонке «Новая» |
| Заявка в команду принята/отклонена | Уведомление участнику, обновление блока «Моя команда» | При принятии назначается роль и дата вступления |
| Добавлен новый документ в идее (Bitrix Диск) | Уведомление стейкхолдеру и исполнителю | В журнале идеи фиксируется файл и автор |
| Просрочена задача Bitrix | Уведомление исполнителю и координатору, отметка «Просрочено» в отчёте | Повторные напоминания настраиваются координатором |
| Задача завершена | Отражается в отчёте «Активность команды», событие в журнале идеи | Обновляются счётчики прогресса |
| Готовность пакета документов (ПК/НТК) подтверждена координатором в Bitrix24 | В Bitrix24 формируется чек-лист финальной проверки, отправляется внутреннее уведомление; на административной доске создаётся задача «Проверить комплект документов и утвердить отправку» (исполнитель — стейкхолдер) | Процесс остается внутри Bitrix24; задача в колонке «Новая» |
| Перевод идеи в «Архив» | Фиксация итога (ПК/НТК), закрытие доступа к задачам, уведомление команды | Дальнейшее сопровождение ведётся вне платформы |
