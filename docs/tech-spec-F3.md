# Техническое задание — Фаза 3 (Informika)

## 1. Назначение и цели

- Публичная витрина результатов и активности: события, одобренные идеи и опубликованные кейсы в режиме «только просмотр».
- Базовая расширенная аналитика (дашборды + выгрузки Excel) без дублирования логики Bitrix24.
- Автоматизация коммуникаций по событиям (подтверждения регистрации, напоминания) и прозрачные статусы для управленческой отчётности.

## 2. Область охвата

### Включено

1. Публичные страницы идей/кейсов в режиме «только просмотр» с минималистичными карточками, без действий по команде или модерации.
2. Публичный календарь событий (онлайн/офлайн), регистрация на событие, рассылка напоминаний.
3. Расширенная аналитика:
   - публичные метрики (агрегированные, без персональных данных);
   - личная статистика по ролям (участник, стейкхолдер, руководство) и операционная статистика координатора в Bitrix24;
   - выгрузки Excel по текущим фильтрам на данных портала + Bitrix24 (выгрузка предназначена для просмотра и отчётности).
4. Публичный каталог команд/результатов (для одобренных идей и опубликованных кейсов) со ссылками на публичные материалы (презентации, демо).
5. Стабилизация интеграции с Bitrix24: портал хранит у себя необходимые данные и регулярно обновляет их; при временных сбоях Bitrix24 портал показывает последние успешные данные и продолжает работать.

### Исключено

- Управление командами и ролями: остаётся в Bitrix24 (портал отображает только текущий состав).
- Процедуры ПК/НТК и любые ведомственные согласования — только в Bitrix24.
- Внутренняя отчётность Bitrix24/кастомные отчёты внутри Bitrix24 — используем стандартные отчёты Bitrix24 и простые выгрузки; интеграции с внешними системами аналитики вне объёма.
- Подписки на уведомления (топики/каналы), счётчик/история уведомлений, дайджест — перенос на следующий релиз.
- Разграничение по секретности/гостайне — вне объёма Ф3.

## 3. Роли и права (дополнения)

- Стейкхолдер: просматривает дашборды/статистику своего направления, архивирует проекты и публикует кейсы своего направления, управляет событиями своих направлений.
- Координатор: управляет календарём событий, утверждает публичные материалы, запускает рассылки.
- Участник/Автор: просматривает личную статистику, может публиковать публичные материалы своей идеи (по разрешению координатора), регистрируется на события.
- Гость: регистрируется на события, читает публичные идеи и опубликованные кейсы.
- Руководство: просматривает сводную аналитику программы в режиме «только просмотр» и выгрузки Excel.

## 4. Жизненный цикл (дополнения Ф3)

- При переводе идеи в «Архив» автоматически создаётся черновик «Кейса» — публичной карточки результата по этой архивной идее/проекту. По умолчанию черновик не отображается в публичной витрине.
- «Архивировать» и «Публиковать как кейс» — разные решения:
  - «Архивировать» закрывает проект внутри программы и закрывает рабочее пространство в Bitrix24 (см. 5.6).
  - «Публиковать как кейс» управляет видимостью результата на портале и не меняет статус идеи и задачи в Bitrix24 (см. 5.1.1).
- Статусы публикации кейса: Черновик → Опубликовано → Снято с публикации (повторная публикация возможна).
- События: Черновик → Опубликовано → Завершено → Архив.

## 5. Функциональные блоки

### 5.1 Публичная витрина

- Каталог витрины разделён на: **Идеи** (только `Одобрена`) и **Кейсы** (только опубликованные кейсы — результаты архивных идей/проектов). Архивные идеи без опубликованного кейса не отображаются в публичной витрине.
- Карточка публичной идеи: направление, статус, дата «Опубликовано/Обновлено», команда (текущий состав), краткое описание, материалы (презентация/видео/ссылка на демо), кнопка «Подробнее». Без кнопок управления командой или статусов.
- Дополнительные страницы (см. interface-elements.md, разд. 14): публичный каталог кейсов (плитка карточек с фильтрами по направлению/тегам; в публичной витрине отображаются только опубликованные кейсы), страница кейса в режиме «только просмотр» с блоками «Описание», «Кратко о результате», «Материалы», «Команда» и кнопкой «Перейти».

#### 5.1.1 «Кейс» (публичная карточка результата архивной идеи/проекта)

- **Назначение:** публичная витрина результата по завершённой идее/проекту; используется для PR/демонстраций и демонстрации эффекта программы.
- **Модель:** на каждую архивную идею/проект создаётся один кейс‑черновик (см. 5.6). Архивация и публикация кейса — разные решения.
- **Публичность:**
  - гостю доступен только опубликованный кейс;
  - черновик и снятые с публикации кейсы доступны только авторизованным ролям (координатор/стейкхолдер).
- **Статусы публикации и переходы (управляет стейкхолдер):**
  - `Черновик` → `Опубликовано` (кнопка `Опубликовать`).
  - `Опубликовано` → `Снято с публикации` (кнопка `Снять с публикации`, обязательна причина).
  - `Снято с публикации` → `Опубликовано` (повторная публикация).
- **Что попадает в публичную витрину:** только кейсы со статусом публикации `Опубликовано`. Архивная идея без опубликованного кейса гостю не показывается.
- **Команда в кейсе:** отображается текущий состав команды (данные из Bitrix24). Для гостя по умолчанию — роль + инициалы (или маска при необходимости), для авторизованных — ФИО + роль.
- **Действия по ролям:**
  - Стейкхолдер: архивирует проект (см. 5.6), заполняет/редактирует карточку результата, публикует/снимает кейс, отвечает за корректность и публичность.
  - Координатор: помогает собрать материалы и оформить черновик, может инициировать правки и проверку, но не публикует и не снимает кейс.
  - Участник/Автор: предлагает материалы/цифры/факты через карточку идеи/Bitrix24; напрямую кейс не публикует.

##### Форма «Публиковать как кейс» (заполнение карточки результата)

Форма доступна в карточке архивной идеи/проекта в блоке «Кейс». Можно сохранить как черновик; публиковать и снимать с публикации может только стейкхолдер.

**1) Основное**

- `Название кейса`\* (предзаполняется названием идеи, можно изменить).
- `Направление` (предзаполняется, без редактирования).
- `Короткое описание результата`\* (1–3 предложения, что получилось в итоге).
- `Дата завершения` (предзаполняется датой архивации, можно уточнить).
- `Теги` (для фильтров витрины; необязательно).

**2) Проблема и цель**

- `Проблема/контекст`\* (что было не так, почему проект запускали).
- `Цель`\* (какой результат ожидали).

**3) Решение**

- `Что сделали`\* (краткое описание реализованного решения).
- `Где внедрено` (подразделение/площадка/процесс; необязательно).

**4) Результаты**

- `Итог`\* (что изменилось/какой эффект получен).
- `Показатели результата` (таблица строк):
  - `Показатель` (например, «время обработки заявки», «экономия», «рост охвата»),
  - `Значение` и `Единица измерения`,
  - `Сравнение` («до/после» или «план/факт»),
  - `Период` (за какой срок измеряли),
  - `Подтверждение` (ссылка на отчёт/замер/протокол).
  - Правило: для публикации кейса либо заполнить минимум один показатель, либо отметить переключатель `Нет измеримых данных`.
- `Следующие шаги` (необязательно).

**5) Материалы**

- `Материалы`\* — список ссылок (минимум 1):
  - `Тип` (презентация/видео/демо/публикация/документ),
  - `Название`\*,
  - `Ссылка`\*,
  - `Видимость` (`Публично` / `Только авторизованным`).
  - Правило: для публикации кейса минимум один материал должен быть с видимостью `Публично`.

**6) Команда**

- Блок заполняется автоматически из Bitrix24 (только просмотр): участники и их роли в проекте.

**7) Проверка перед публикацией (обязательно)**

- чекбокс `Материалы и текст не содержат персональных/закрытых данных`.
- чекбокс `Публичные ссылки открываются без доступа к Bitrix24`.
- чекбокс `Кейс согласован к публикации`.

**Кнопки**

- `Сохранить черновик` (стейкхолдер, координатор).
- `Опубликовать` (только стейкхолдер).
- `Снять с публикации` (только стейкхолдер; обязательное поле «Причина снятия»).

### 5.2 События

- Календарь обязателен: вид «месяц/неделя» + список; фильтры направление, формат (Онлайн/Офлайн), статус (Будущие/Прошедшие).
- Карточка события: дата/время, формат, место/ссылка, спикеры, материалы после события (одна ссылка/кнопка). Кнопки: «Зарегистрироваться», «Добавить в календарь» (чтобы событие появилось в личном календаре), после завершения — «Материалы».
- Страница события: краткое описание, программа, спикеры, блок «Материалы» (после завершения), минимальная форма регистрации (ФИО, e-mail, роль) с подтверждением и уведомлением.
- Уведомления: подтверждение регистрации, напоминание за 24/2 часа, пост-фактум с материалами.

### 5.3 Аналитика и отчётность

#### 5.3.1 Общие правила, фильтры и экспорт

- Аналитика в Ф3 — **только просмотр**. Любые решения/переводы статусов/управление командами выполняются в Bitrix24 (портал не дублирует бизнес‑логику).
- Источники данных:
  - Портал: запросы/идеи/кейсы/события/регистрации на события.
  - Bitrix24: статусы задач, дедлайны и факты просрочки, состав команд, связи «идея ↔ проект/группа ↔ задачи».
- Обновление и актуальность:
  - Портал обновляет данные из Bitrix24 **каждые 30 минут**. Если Bitrix24 временно недоступен, портал повторяет попытку позже и показывает последние успешно обновлённые данные.
  - На каждом дашборде показывать дату и время последнего успешного обновления данных (по Москве).
- Временная зона и формат дат:
  - Все даты и время считаются по московскому времени.
  - Форматы: дата `ДД.ММ.ГГГГ`, дата‑время `ДД.ММ.ГГГГ HH:MM`.
- Общие фильтры (если для виджета не указано иное):
  - `Период`: `7 дней` / `30 дней` / `Текущий квартал` / `Год` / `Произвольный диапазон`.
  - `Направление`: `Все` или одно значение из справочника направлений.
  - `Статус` (для таблиц): множественный выбор статусов сущности.
- Экспорт:
  - Экспорт доступен только в Excel. **CSV и PDF не используются.**
  - Экспорт доступен только авторизованным пользователям; на публичной странице метрик (п. 5.3.2) кнопка экспорта отсутствует.
  - Экспорт формируется по текущим фильтрам страницы/раздела и содержит:
    - вкладку «Описание»: когда сформирован файл, какие фильтры выбраны и что означает каждая метрика;
    - листы с табличными данными, на основе которых построены виджеты (см. ниже по каждому дашборду).

#### 5.3.2 Публичные метрики (без авторизации)

- Назначение: внешний «срез» по программе без персональных данных (только агрегаты).
- Правило приватности: в публичных метриках **не отображаются ФИО, e‑mail и внутренние идентификаторы пользователей**; идеи в статусе `Черновик` не учитываются.
- Экран: `Публичная статистика` (одна страница, только просмотр).
- Виджеты и метрики:
  - **Ключевые показатели:**
    - `Активные запросы` = количество запросов со статусом `На сборе идей` на момент обновления данных.
    - `Новые запросы за период` = количество запросов, опубликованных в выбранный период.
    - `Закрытые запросы за период` = количество запросов, закрытых в выбранный период.
    - `Подано идей за период` = количество идей, отправленных на рассмотрение в выбранный период.
    - `Доля одобрения за период` = доля одобренных идей среди всех идей, по которым принято решение в выбранный период (одобрено + отклонено).
    - `Идеи в реализации` = количество идей в статусе `Одобрена` на момент обновления данных.
    - `Завершено проектов за период` = количество идей, переведённых в `Архив` в выбранный период (по дате перехода в `Архив` в истории статусов идеи).
  - **Графики:**
    - `Воронка идей (текущее состояние)` — количество идей по статусам: `На модерации`, `На доработке`, `На рассмотрении`, `Одобрена`, `Отклонена`, `Архив` (на момент обновления данных).
    - `Динамика подачи идей` — количество идей по датам подачи (по дням для периода до 30 дней, иначе по неделям).
    - `Динамика решений` — количество решений по датам принятия решения (разделение: `Одобрена`/`Отклонена`).
    - `Сроки прохождения (модерация/решение)`:
      - `Срок модерации` = время от входа в статус `На модерации` до выхода из него (`На доработке` или `На рассмотрении`) по данным истории статусов.
      - `Срок до решения` = время от входа в статус `На рассмотрении` до финального решения (`Одобрена`/`Отклонена`) по данным истории статусов.
      - На графике показывать: `среднее`, `медиана`, `количество рассмотренных идей` за период (учитываются идеи, у которых соответствующий этап завершился в периоде).
    - `Распределение по направлениям` — два ряда: `Подано идей` и `Одобрено идей` за период.
    - `События: регистрации` — количество регистраций за период с разрезом по формату события (`Онлайн`/`Офлайн`) и по направлениям.
  - **Таблицы (топ‑листы):**
    - `Топ запросов по количеству идей` (за период): `Название запроса`, `Направление`, `Идей подано`.
    - `Топ событий по регистрациям` (за период): `Название`, `Дата/время`, `Формат`, `Регистраций`.

#### 5.3.3 Личная статистика участника (ЛК)

- Доступ: участник видит только свои данные (свои идеи, свои регистрации, свои задачи Bitrix24).
- Экран: `ЛК → Статистика` (только просмотр, с переходами в списки/карточки).
- Фильтры: `Период` (как в п. 5.3.1), `Направление` (по направлению запроса/идеи).
- Виджеты и метрики:
  - **Счётчики (мои идеи):**
    - `Мои идеи всего` — количество идей, где я указан автором.
    - `В работе` — количество моих идей в статусах `Черновик`, `На модерации`, `На доработке`, `На рассмотрении`, `Одобрена`.
    - `На модерации` — сколько моих идей сейчас проверяет координатор.
    - `На доработке` — сколько моих идей возвращены с замечаниями.
    - `На рассмотрении` — сколько моих идей ожидают решения стейкхолдера.
    - `Одобрено за период` — сколько моих идей было одобрено в выбранный период.
    - `Отклонено за период` — сколько моих идей было отклонено в выбранный период.
    - `В реализации` — сколько моих идей сейчас в статусе `Одобрена`.
    - `Завершено` — сколько моих идей переведены в `Архив`.
  - **Счётчики (мои события):**
    - `Регистрации: будущие` — количество моих регистраций на события, которые ещё не начались.
    - `Регистрации: прошедшие` — количество моих регистраций на события, которые уже прошли.
  - **Графики:**
    - `Мои идеи по статусам` — распределение моих идей по статусам.
    - `Динамика подачи моих идей` — сколько идей я подал по датам (по дням/неделям, как в п. 5.3.2).
    - `Динамика решений по моим идеям` — сколько моих идей было одобрено/отклонено по датам решения.
    - `Срок до решения (мои идеи)` — средний и медианный срок от подачи идеи до решения + количество рассмотренных идей за период.
    - `Возвраты на доработку` — среднее количество возвратов на доработку на одну идею за период.
  - **Bitrix24 (задачи по моим проектам, только просмотр):**
    - `Открытые задачи` — количество задач, где я назначен ответственным и которые ещё не закрыты.
    - `Просроченные задачи` — количество моих задач, у которых срок выполнения уже прошёл, но задача не закрыта.
    - `Задачи на 7 дней` — количество моих задач со сроком выполнения в ближайшие 7 дней (и задача не закрыта).
    - `Закрыто задач за период` — сколько моих задач было закрыто в выбранный период.
    - `Топ-5 просроченных задач` — название, проект/идея, срок, дней просрочки, ссылка в Bitrix24.
  - **Кейсы:**
    - `Кейсы с моим участием` — количество кейсов со статусом публикации `Опубликовано`, где я в команде.
  - Экспорт в Excel включает вкладки: «Описание», «Мои идеи», «Мои события», «Мои задачи Bitrix24», «Кейсы».

#### 5.3.4 Личная статистика стейкхолдера (портфель направления)

- Доступ: стейкхолдер видит статистику по своим направлениям и своим запросам/идеям (только просмотр), без возможности менять статусы в портале.
- Экран: `ЛК стейкхолдера → Аналитика` (только просмотр).
- Фильтры:
  - `Период` (как в п. 5.3.1).
  - `Направление` (только направления, закреплённые за стейкхолдером).
  - `Запрос`: `Все` (по умолчанию) или один конкретный запрос.
- Виджеты и метрики:
  - **Ключевые показатели:**
    - `Активные запросы` = запросы со статусом `На сборе идей` в выбранном направлении.
    - `Новые идеи за период` = количество идей, поданных в выбранный период по запросам направления.
    - `Идеи на рассмотрении` = количество идей в статусе `На рассмотрении` (текущая очередь на решение).
    - `Решений за период` = количество идей, по которым принято решение в выбранный период (разделение `Одобрена`/`Отклонена`) + `Доля одобрения`.
    - `Проекты в реализации` = количество идей в статусе `Одобрена`.
    - `Проекты завершены за период` = количество идей, переведённых в `Архив` в периоде.
    - `Просроченных задач в проектах` = количество задач Bitrix24 по проектам направления, у которых срок выполнения уже прошёл, но задача не закрыта.
    - `Опубликовано кейсов` = количество кейсов со статусом публикации `Опубликовано` по направлению.
    - `События: будущие` / `События: проведено за период` / `Регистраций за период` (по событиям выбранного направления).
  - **Графики:**
    - `Воронка идей направления (текущее состояние)` — количество идей по статусам: `На модерации`, `На доработке`, `На рассмотрении`, `Одобрена`, `Отклонена`, `Архив`.
    - `Динамика идей` — по датам подачи идей (по дням/неделям по правилу из п. 5.3.2).
    - `Динамика решений` — по датам принятия решения (`Одобрена`/`Отклонена`).
    - `Сроки прохождения (по направлению)` — как в п. 5.3.2, но только по выбранному направлению (показывать `среднее`, `медиана`, `количество рассмотренных идей`).
    - `События: регистрации` — регистрации за период по форматам/событиям.
  - **Таблицы:**
    - `Запросы направления` (за период): `Название`, `Статус`, `Дедлайн`, `Идей подано`, `Решений`, `Одобрено`, `Отклонено`.
    - `Очередь на рассмотрение` (текущее): `Идея`, `Запрос`, `Дата поступления на рассмотрение`, `Дней в статусе`, `Ссылка в Bitrix24`.
    - `Проекты с рисками` (текущее): `Проект/идея`, `Открытых задач`, `Просроченных задач`, `Ближайший дедлайн`, `Ссылка в Bitrix24`.
    - `Топ кейсов` (текущее): `Название кейса`, `Дата публикации`, `Ссылка`.
  - Экспорт в Excel включает вкладки: «Запросы», «Воронка идей», «Сроки прохождения идей», «Задачи Bitrix24 (агрегаты)», «События», «Кейсы».

#### 5.3.5 Координатор — операционная статистика в Bitrix24

- Принцип: координаторская операционная статистика строится **в Bitrix24** (стандартные отчёты/представления смарт‑процессов); портал не дублирует управление и не вводит отдельные «управляющие» отчёты.
- Обязательные представления/виджеты в Bitrix24 (для просмотра, действия — по правам Bitrix24):
  - `Очередь модерации` (смарт‑процесс «Идея»): количество идей в статусах `На модерации` и `На доработке`, с группировкой по времени ожидания в статусе: `до 1 суток`, `1–2 суток`, `2–3 суток`, `более 3 суток`.
  - `Производительность модерации` (за период): сколько идей вышло из `На модерации` (в `На доработке`/`На рассмотрении`), средний/медианный срок модерации, доля возвратов на доработку.
  - `Контроль дедлайнов запросов` (смарт‑процесс «Запрос»): активные запросы с дедлайном в ближайшие `7 дней` и просроченные запросы (если допускается ручное закрытие).
  - `Интеграции (виджет состояния)`:
    - дата и время последнего успешного обновления данных на портале,
    - количество ошибок обновления за последние 24 часа,
    - средняя задержка обновления (в минутах): как быстро изменения в Bitrix24 появляются на портале.

#### 5.3.6 Руководство — сводная статистика программы

- Доступ: роль `Руководство` (и при необходимости координатор) видит сводный дашборд по всем направлениям (только просмотр).
- Экран: `Админ/Аналитика → Сводка` (только просмотр) + `Экспорт в Excel`.
- Фильтры: `Период`, `Направление` (`Все`/одно), `Стейкхолдер` (`Все`/один).
- Виджеты и метрики:
  - **Ключевые показатели:**
    - `Активные запросы` (сейчас), `Новые запросы` (за период), `Закрытые запросы` (за период).
    - `Подано идей` (за период), `Решений` (за период), `Доля одобрения` (за период).
    - `Проекты в реализации` (сейчас), `Завершено проектов` (за период).
    - `Опубликовано кейсов` (сейчас, статус публикации `Опубликовано`), `Новых кейсов` (за период, впервые опубликованных).
    - `Будущие события` (сейчас), `Проведено событий` (за период), `Регистраций` (за период).
    - `Просроченных задач` (сейчас) — по всем проектам Bitrix24, связанным с идеями в статусе `Одобрена`.
  - **Графики:**
    - `Воронка идей (текущее состояние)` — как в п. 5.3.2 (по всем направлениям либо по выбранному).
    - `Динамика подачи идей` и `Динамика решений` — как в п. 5.3.2.
    - `Сроки прохождения` — как в п. 5.3.2 (показывать `среднее`, `медиана`, `количество рассмотренных идей`).
    - `Распределение по направлениям` — подано/одобрено/архив за период.
    - `Риски проектов` — динамика количества просроченных задач и количества проектов с просрочками (за период, по неделям).
  - **Таблицы:**
    - `Топ направлений` (за период): `Направление`, `Идей подано`, `Одобрено`, `В реализации`, `Архив`, `Кейсы опубликованы`, `События проведено`.
    - `Топ запросов` (за период): как в п. 5.3.2, + `Одобрено`/`Отклонено`.
    - `Проекты с рисками` (текущее): как в п. 5.3.4, + `Направление`, `Стейкхолдер`.
  - Экспорт в Excel включает вкладки: «Показатели», «По направлениям», «Запросы», «Воронка идей», «Сроки прохождения идей», «Задачи Bitrix24», «События», «Кейсы».

### 5.4 Интеграции и надёжность

#### 5.4.1 Кэширование данных Bitrix24 (обязательно)

- Портал хранит у себя копию данных, которые нужны для витрины и статистики, чтобы страницы открывались быстро и не «падали» при временных сбоях Bitrix24.
- Что именно кэшируем:
  - справочники (направления, перечень запросов и идей, статусы, связи «идея ↔ проект/задачи»);
  - состав команд по проектам (кто состоит в команде и с какой ролью);
  - данные для контроля сроков в проектах (сколько задач открыто, сколько просрочено, какие сроки ближайшие).
- Как обновляем:
  - оперативные данные (статусы, сроки задач, состав команд) — каждые 30 минут;
  - справочники — минимум 1 раз в сутки, а также по ручной команде координатора «Обновить данные».
- Актуальность на экране:
  - на каждой странице статистики показывать дату и время последнего успешного обновления данных;
  - если данные устарели более чем на сутки — показывать предупреждение, что данные могут быть неактуальны.
- Поведение при недоступности Bitrix24:
  - портал показывает последние сохранённые данные и сообщение «Bitrix24 временно недоступен, показаны данные на момент последнего обновления»;
  - кнопка «Повторить» запускает повторную попытку обновления;
  - если данных ещё нет (например, при первом запуске) — показывать пустое состояние и сообщение об ошибке без падения страницы.

#### 5.4.2 Почтовые уведомления

- Письма отправляются только по событиям: подтверждение регистрации, напоминания, письмо с материалами после события.

#### 5.4.3 Журнал интеграций

- Координатор видит список ошибок интеграции (дата/время, что не удалось получить, краткая причина) и может повторить обновление.

### 5.5 Профили (навыки, интересы, участие в проектах)

- Расширенные поля профиля: навыки (теги), интересы (направления/темы), участие в проектах (список кейсов с ролями).
- Поиск и фильтры по профилям: поиск по имени, навыкам, интересам; фильтр по роли/участию в проектах; выдача — карточки профилей с краткой информацией и кнопкой «Подробнее».
- Отображение в профиле: блок «Навыки и интересы», блок «Участие в проектах» (список идей/кейсов с ролями), кнопка «Редактировать» для владельца.

### 5.6 Архивация идей (преобразование в проекты)

- Архивация — это «закрыть проект внутри программы»: перевести идею/проект в статус `Архив`.
- Инициатор: стейкхолдер (только по своим направлениям). Условие: идея в статусе `Одобрена`.
- При архивации стейкхолдер обязан заполнить `Сопроводительное сообщение` (видно команде и координатору): краткий итог, что получилось, где смотреть материалы/демо; если проект закрыт без результата — указать причину.
- Что происходит при архивации:
  - в Bitrix24 закрывается рабочее пространство проекта (группа/проект переводится в режим завершения/архива; новые задачи и процессы по проекту больше не запускаются);
  - портал фиксирует минимальную статистику на дату архивации (для аналитики и истории проекта): длительность (от одобрения до архивации), состав команды (количество участников и роли), задачи Bitrix24 (всего/закрыто/открыто, из открытых — просрочено), количество материалов и сколько из них помечены как публичные;
  - автоматически создаётся кейс со статусом публикации `Черновик` (см. 5.1.1). По умолчанию он не отображается в публичной витрине.
- Отображение:
  - в кабинете стейкхолдера/координатора архивная идея/проект остаётся доступной для просмотра вместе с сопроводительным сообщением, ссылками на материалы и ссылкой на Bitrix24;
  - в публичной витрине архивные идеи/проекты не показываются — публично отображаются только кейсы со статусом публикации `Опубликовано`.

### 5.7 Календарь и напоминания (обязательный элемент)

- Календарь событий обязателен: вид «месяц/неделя» + список; фильтры направление/формат/статус.
- Напоминания: автоматические уведомления за 24/2 часа до начала события, ссылка на подключение/адрес, пост-анонс с материалами после события.
- Анонсы и ссылки: карточка события содержит ссылку на трансляцию или адрес офлайна, кнопку «Добавить в календарь» и ссылку на материалы после завершения.

## 6. UX/Интерфейс

- Базируется на принципах минималистичных карточек из Ф2: одна главная кнопка «Подробнее/Перейти», без индикаторов заполненности команды, только текущий состав и датированные бейджи («Опубликовано/Обновлено ДД.ММ.ГГГГ» или «Просрочено» для событий).
- Портфель идей стейкхолдера: карточки (одобренные/архивные) + блок фильтр-чипов, быстрые ссылки на материалы/Bitrix24.
- События: лёгкая форма регистрации (ФИО, e-mail, роль), подтверждение на экране + уведомление.

## 7. Данные и интеграции

- Новые сущности:
  - Кейс: ссылка на исходную архивную идею/проект, название, направление, теги, описание (проблема/цель/решение/итог), показатели результата, материалы (ссылки с признаком публичности), команда (берётся из Bitrix24), статус публикации (`Черновик`/`Опубликовано`/`Снято с публикации`), дата публикации, дата снятия с публикации, причина снятия (обязательна при снятии).
  - Событие: название, дата и время, формат (онлайн/офлайн), место или ссылка, спикеры, материалы, статус события, даты создания/публикации/завершения.
  - Регистрация на событие: событие, пользователь (если авторизован), ФИО, e‑mail, роль участника, дата регистрации.
- Расширения к идее/проекту: дата архивации и сопроводительное сообщение стейкхолдера; связь «идея/проект → кейс» (кейс‑черновик создаётся автоматически при архивации); ссылки на материалы (с признаком публичности).
- Материалы: файлы остаются в Bitrix24 или во внешнем хранилище; на портале храним только ссылки.

## 8. Приёмочные критерии

1. Публичный каталог кейсов доступен без авторизации: в каталоге отображаются только кейсы со статусом публикации `Опубликовано`, они открываются в карточку «только просмотр»; карточки показывают дату «Опубликовано/Обновлено», без кнопок управления командой/решений.
2. События: календарь/список, регистрация, напоминания, материалы после события работают; регистрация гостя фиксируется, есть кнопка «Добавить в календарь» и ссылка на трансляцию или адрес.
3. Аналитика разделена на публичные метрики и личную статистику по ролям (п. 5.3); все дашборды — только просмотр, данные обновляются по расписанию, экспорт доступен только в Excel (без CSV/PDF); внешних подключений к системам аналитики нет.
4. Расширенный профиль (навыки/интересы, участие в проектах) доступен для просмотра/редактирования, поиск/фильтры по профилям работают.
5. При переводе идеи/проекта в `Архив` автоматически создаётся кейс со статусом публикации `Черновик`; опубликовать/снять с публикации может только стейкхолдер, при снятии причина обязательна; гостю доступны только кейсы со статусом публикации `Опубликовано`, публикация/снятие не меняет статус идеи и задачи в Bitrix24.

## 9. Зависимости и предпосылки

- Доступ к Bitrix24 для чтения данных о задачах, командах и материалах (чтение, без управления).
- Настроенная отправка e‑mail для писем по событиям.
- Внешние системы аналитики не требуются; используются внутренние дашборды и выгрузки Excel.
