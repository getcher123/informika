# Техническое задание — Фаза 3 (Informika)

## 1. Назначение и цели

- Публичный портфель проектов: события, одобренные идеи и опубликованные кейсы в режиме «только просмотр».
- Базовая аналитика (блоки статистики + выгрузки Excel) без дублирования логики Bitrix24.
- Автоматизация коммуникаций по событиям (подтверждения регистрации, напоминания) и прозрачные статусы для управленческой отчётности.

## 2. Область охвата

### Включено

1. Публичные страницы идей/кейсов в режиме «только просмотр» с минималистичными карточками, без действий по команде или модерации.
2. Публичный календарь событий (онлайн/офлайн), регистрация на событие, рассылка напоминаний.
3. Расширенная аналитика:
   - публичные метрики (агрегированные, без персональных данных);
   - личная статистика по ролям (участник, стейкхолдер, руководство) и операционная статистика координатора в Bitrix24;
   - выгрузки Excel по текущим фильтрам на данных портала + Bitrix24 (выгрузка предназначена для просмотра и отчётности).
4. Публичный каталог команд/результатов (для одобренных идей и опубликованных кейсов) со ссылками на публичные материалы (презентации, демо).
5. Стабилизация интеграции с Bitrix24: портал хранит у себя необходимые данные и регулярно обновляет их; при временных сбоях Bitrix24 портал показывает последние успешные данные и продолжает работать.

### Исключено

- Управление командами и ролями: остаётся в Bitrix24 (портал отображает только текущий состав).
- Процедуры ПК/НТК и любые ведомственные согласования — только в Bitrix24.
- Внутренняя отчётность Bitrix24/кастомные отчёты внутри Bitrix24 — используем стандартные отчёты Bitrix24 и простые выгрузки; интеграции с внешними системами аналитики вне объёма.
- Подписки на уведомления (топики/каналы), счётчик/история уведомлений, дайджест — перенос на следующий релиз.
- Разграничение по секретности/гостайне — вне объёма Ф3.

## 3. Роли и права (дополнения)

- Стейкхолдер: просматривает статистику своего направления, архивирует проекты и публикует кейсы своего направления, управляет событиями своих направлений.
- Координатор: управляет календарём событий, утверждает публичные материалы, запускает рассылки.
- Участник/Автор: просматривает личную статистику, может публиковать публичные материалы своей идеи (по разрешению координатора), регистрируется на события.
- Гость: регистрируется на события, читает публичные идеи и опубликованные кейсы.
- Руководство: просматривает сводную аналитику программы в режиме «только просмотр» и выгрузки Excel.

## 4. Жизненный цикл (дополнения Ф3)

- При переводе идеи в «Архив» автоматически создаётся черновик «Кейса» — публичной карточки результата по этой архивной идее/проекту. По умолчанию черновик не отображается в публичном портфеле проектов.
- «Архивировать» и «Публиковать как кейс» — разные решения:
  - «Архивировать» закрывает проект внутри программы и закрывает рабочее пространство в Bitrix24 (см. 5.6).
  - «Публиковать как кейс» управляет видимостью результата на портале и не меняет статус идеи и задачи в Bitrix24 (см. 5.1.1).
- Статусы публикации кейса: Черновик → Опубликовано → Снято с публикации (повторная публикация возможна).
- События: Черновик → Опубликовано → Завершено/Отменено → Архив.

## 5. Функциональные блоки

### 5.1 Портфель проектов

- Каталог портфеля проектов разделён на: **Идеи** (только `Одобрена`) и **Кейсы** (только опубликованные кейсы — результаты архивных идей/проектов). Архивные идеи без опубликованного кейса не отображаются в публичном портфеле проектов.
- Карточка публичной идеи: направление, статус, дата «Опубликовано/Обновлено», команда (текущий состав), краткое описание, материалы (презентация/видео/ссылка на демо), кнопка «Подробнее». Без кнопок управления командой или статусов.
- Дополнительные страницы (см. interface-elements.md, разд. 14): публичный каталог кейсов (плитка карточек с фильтрами по направлению; в публичном портфеле проектов отображаются только опубликованные кейсы), страница кейса в режиме «только просмотр» с блоками «Описание», «Кратко о результате», «Материалы», «Команда» и кнопкой «Перейти».

#### 5.1.1 «Кейс» (публичная карточка результата архивной идеи/проекта)

- **Назначение:** публичная карточка результата по завершённой идее/проекту в «Портфеле проектов»; используется для PR/демонстраций и демонстрации эффекта программы.
- **Модель:** на каждую архивную идею/проект создаётся один кейс‑черновик (см. 5.6). Архивация и публикация кейса — разные решения.
- **Публичность:**
  - гостю доступен только опубликованный кейс;
  - черновик и снятые с публикации кейсы доступны только авторизованным ролям (координатор/стейкхолдер).
- **Статусы публикации и переходы (управляет стейкхолдер):**
  - `Черновик` → `Опубликовано` (кнопка `Опубликовать`).
  - `Опубликовано` → `Снято с публикации` (кнопка `Снять с публикации`, обязательна причина).
  - `Снято с публикации` → `Опубликовано` (повторная публикация).
- **Что попадает в публичный портфель проектов:** только кейсы со статусом публикации `Опубликовано`. Архивная идея без опубликованного кейса гостю не показывается.
- **Команда в кейсе:** отображается текущий состав команды (данные из Bitrix24). Для гостя по умолчанию — роль + инициалы (или маска при необходимости), для авторизованных — ФИО + роль.
- **Действия по ролям:**
  - Стейкхолдер: архивирует проект (см. 5.6), заполняет/редактирует карточку результата, публикует/снимает кейс, отвечает за корректность и публичность.
  - Координатор: помогает собрать материалы и оформить черновик, может инициировать правки и проверку, но не публикует и не снимает кейс.
  - Участник/Автор: предлагает материалы/цифры/факты через карточку идеи/Bitrix24; напрямую кейс не публикует.

##### Форма «Публиковать как кейс» (заполнение карточки результата)

Форма доступна в карточке архивной идеи/проекта в блоке «Кейс». Можно сохранить как черновик; публиковать и снимать с публикации может только стейкхолдер.

**1) Основное**

- `Название кейса`\* (предзаполняется названием идеи, можно изменить).
- `Направление` (предзаполняется, без редактирования).
- `Короткое описание результата`\* (1–3 предложения, что получилось в итоге).
- `Дата завершения` (предзаполняется датой архивации, можно уточнить).

**2) Проблема и цель**

- `Проблема/контекст`\* (что было не так, почему проект запускали).
- `Цель`\* (какой результат ожидали).

**3) Решение**

- `Что сделали`\* (краткое описание реализованного решения).
- `Где внедрено` (подразделение/площадка/процесс; необязательно).

**4) Результаты**

- `Итог`\* (что изменилось/какой эффект получен).
- `Показатели результата` (таблица строк):
  - `Показатель` (например, «время обработки заявки», «экономия», «рост охвата»),
  - `Значение` и `Единица измерения`,
  - `Сравнение` («до/после» или «план/факт»),
  - `Период` (за какой срок измеряли),
  - `Подтверждение` (ссылка на отчёт/замер/протокол).
  - Правило: для публикации кейса либо заполнить минимум один показатель, либо отметить переключатель `Нет измеримых данных`.
- `Следующие шаги` (необязательно).

**5) Материалы**

- `Материалы`\* — список ссылок (минимум 1):
  - `Тип` (презентация/видео/демо/публикация/документ),
  - `Название`\*,
  - `Ссылка`\*,
  - `Видимость` (`Публично` / `Только авторизованным`).
  - Правило: для публикации кейса минимум один материал должен быть с видимостью `Публично`.

**6) Команда**

- Блок заполняется автоматически из Bitrix24 (только просмотр): участники и их роли в проекте.

**7) Проверка перед публикацией (обязательно)**

- чекбокс `Материалы и текст не содержат персональных/закрытых данных`.
- чекбокс `Публичные ссылки открываются без доступа к Bitrix24`.
- чекбокс `Кейс согласован к публикации`.

**Кнопки**

- `Сохранить черновик` (стейкхолдер, координатор).
- `Опубликовать` (только стейкхолдер).
- `Снять с публикации` (только стейкхолдер; обязательное поле «Причина снятия»).

### 5.2 События

- Календарь обязателен и является частью публичной витрины (видно гостю), при этом управление событиями выполняют координатор/стейкхолдеры.
- Важно: в интерфейсе используется фильтр `Будущие/Прошедшие` — это **не статус**, а группировка по времени проведения.

#### 5.2.1 Статусы и видимость (без двойственности)

- Статусы события:
  - `Черновик` — событие видно только координатору и стейкхолдерам (в личном кабинете).
  - `Опубликовано` — событие видно всем (публичный календарь).
  - `Завершено` — событие прошло (статус выставляется автоматически по времени окончания события); видно всем.
  - `Отменено` — событие отменено организатором; видно всем, регистрация закрыта.
  - `Архив` — событие скрыто из публичного календаря (для истории/служебных целей).
- Переходы:
  - `Черновик` → `Опубликовано` (публикация).
  - `Опубликовано` → `Отменено` (отмена) или `Архив` (скрыть/убрать).
  - `Опубликовано` → `Завершено` (автоматически после окончания).
  - `Завершено/Отменено` → `Архив` (для очистки публичных списков, по решению координатора).
- Фильтр `Будущие/Прошедшие`:
  - `Будущие` — события со статусами `Опубликовано`/`Отменено`, у которых время окончания ещё не наступило.
  - `Прошедшие` — события со статусом `Завершено` (и отменённые, если время уже прошло).

#### 5.2.2 Поля события и правила заполнения

- Обязательные поля:
  - `Название события`.
  - `Направление` (одно из направлений программы или значение `Общее`).
  - `Формат`: `Онлайн` или `Офлайн`.
  - `Дата и время начала`, `Дата и время окончания` (по Москве).
- Поля по формату:
  - Для `Онлайн`: `Ссылка на трансляцию/подключение` (одна ссылка).
  - Для `Офлайн`: `Адрес проведения` (одна строка).
- Дополнительные поля (необязательные, но рекомендуемые):
  - `Краткое описание` (1–3 абзаца).
  - `Программа` (список пунктов/тайминг).
  - `Спикеры` (список: ФИО + должность/роль).
  - `Ограничение мест` (число) — если задано, регистрация закрывается при достижении лимита.
  - `Регистрация до` (дата‑время) — если задано, после этого момента кнопка регистрации скрывается.
  - `Материалы` (ссылки) — добавляются после события.
- Валидации:
  - окончание позже начала;
  - для онлайн‑события ссылка обязательна, для офлайн‑события обязателен адрес.

#### 5.2.3 Публичный календарь (виды/фильтры/карточки)

- Виды: `Календарь (месяц/неделя)` и `Список`.
- Фильтры: `Когда` (`Будущие/Прошедшие`), `Формат` (`Онлайн/Офлайн`), `Направление`, `Поиск по названию`.
- Сортировка:
  - `Будущие` — по дате/времени начала (от ближайших к дальним).
  - `Прошедшие` — по дате/времени начала (от недавних к более старым).
- Карточка события (в календаре/списке):
  - дата и время (начало–окончание), формат, направление;
  - место/ссылка (в зависимости от формата), спикеры (если заполнены);
  - статусная плашка: `Отменено` (если отменено) или `Завершено` (если прошло);
  - кнопки:
    - `Зарегистрироваться` (только для будущих событий, если регистрация открыта);
    - `Добавить в календарь (.ics)` (только для будущих опубликованных событий);
    - после завершения — `Материалы` (если материалы добавлены).

#### 5.2.4 Страница события и регистрация

- Страница события содержит: описание, программу, спикеров, дату/время, место/ссылку и блок регистрации.
- Форма регистрации (минимальная, доступна гостю):
  - `ФИО`, `E-mail`, `Роль` (например: `Участник`, `Стейкхолдер`, `Сотрудник`, `Гость`).
  - чекбокс согласия на обработку персональных данных (обязательно).
- Поведение:
  - при успешной регистрации показывается экран подтверждения на сайте и отправляется e‑mail;
  - повторная регистрация на одно и то же событие с тем же e‑mail не допускается (показывать сообщение «Вы уже зарегистрированы»);
  - если событие `Отменено`, регистрация недоступна и показывается причина отмены (короткий текст).

#### 5.2.5 Уведомления по событиям (e‑mail)

- Подтверждение регистрации — сразу после отправки формы (с данными события и ссылкой `Добавить в календарь (.ics)`).
- Напоминания — автоматически за 24 часа и за 2 часа до начала (только для будущих событий, если они не отменены).
- Материалы — после завершения события, когда координатор добавит материалы и нажмёт `Разослать материалы`.
- Отмена/перенос — если организатор меняет дату/время или отменяет событие, всем зарегистрированным отправляется уведомление (с обновлёнными данными/причиной).

### 5.3 Аналитика и отчётность

#### 5.3.1 Общие правила, период и экспорт

- Статистика в Ф3 нужна в двух целях:
  - **мотивация и PR** (участник видит свой вклад и опубликованные кейсы);
  - **управление** (стейкхолдер/координатор/руководство видят активность, прогресс и риски).
- Аналитика в Ф3 — **только просмотр**. Любые решения/переводы статусов/управление командами выполняются в Bitrix24 (портал не дублирует бизнес‑логику).
- Источники данных:
  - Портал: запросы, идеи, кейсы, события, регистрации на события.
  - Bitrix24: задачи и сроки по рабочим областям одобренных идей, факты просрочки, текущий состав команды и роли.
- Обновление и актуальность:
  - Портал обновляет данные из Bitrix24 **каждые 30 минут**.
  - На любом экране/блоке, где используются данные Bitrix24 (прогресс, риски, задачи), показывать дату и время последнего успешного обновления (по Москве).
- Временная зона и формат дат:
  - Все даты и время считаются по московскому времени.
  - Форматы: дата `ДД.ММ.ГГГГ`, дата‑время `ДД.ММ.ГГГГ HH:MM`.
- Период:
  - для публичного блока на странице «Банк инноваций» период фиксирован: **последние 7 дней** (без переключателя);
  - для экрана руководства допускается выбор периода: `7 дней` / `30 дней` / `Текущий квартал` / `Год` / `Произвольный диапазон`.
- Экспорт:
  - Экспорт доступен только в Excel. **CSV и PDF не используются.**
  - Экспорт доступен только администратору.
  - Экспорт формируется по текущему контексту (страница/фильтры) и содержит вкладку «Описание»: когда сформирован файл и что означает каждая метрика.

#### 5.3.2 Банк инноваций: блок статистики «За 7 дней» (виден всем)

- Назначение: короткий публичный «срез» по активности (без персональных данных).
- Расположение: верх страницы `Банк инноваций`, **над фильтрами**.
- Период: последние 7 дней (скользящее окно); на экране показывать диапазон дат.
- Приватность: в блоке не отображаются ФИО, e‑mail и любые персональные списки; учитываются только агрегаты. Идеи в статусе `Черновик` не учитываются.
- Метрики блока (3–4 карточки‑счётчика):
  - `Новые запросы` — сколько запросов опубликовано за 7 дней.
  - `Подано идей` — сколько идей подано за 7 дней.
  - `Идей одобрено` — сколько идей переведено в статус `Одобрена` за 7 дней.
  - `Идей отклонено` — сколько идей переведено в статус `Отклонена` за 7 дней.
- Экспорт: отсутствует.

#### 5.3.3 Участник: «Мой вклад» (мотивация участия и PR)

- Цель: дать участнику простой ответ «что я сделал» и «что можно показать публично».
- Расположение: `ЛК → Мои идеи` (вверху страницы, над списком).
- Показатели (без графиков):
  - `Идей предложено` (за всё время; при наличии дополнительно показывать `+N за 7 дней`).
  - `Идей одобрено` (за всё время).
  - `Проектов в реализации` (идеи в статусе `Одобрена`, где участник автор).
  - `Моё место среди активных участников` (за 30 дней).
    - Активные участники: авторы, которые за последние 30 дней **подали или дорабатывали** хотя бы одну идею (кроме `Черновик`).
    - Как считаем место: сортируем активных участников по `баллам активности` (по убыванию), место — позиция в списке.
    - Баллы активности (за 30 дней): `1 балл` за каждую идею, которую участник подал или дорабатывал в период; `+2 балла` за каждую идею участника, которая в период получила статус `Одобрена`.
    - Равенство баллов: выше тот, у кого больше `идей подано/доработано` за 30 дней; далее — больше `одобрено` за 30 дней; далее — больше `идей предложено` за всё время.
    - Отображение: `#<место> из <кол-во активных>`; если участник не активен за 30 дней — показывать `Не в рейтинге` и подсказку «Подайте или обновите идею, чтобы попасть в рейтинг».
- PR‑блок: ниже показывать список `Мои опубликованные кейсы` (название, направление, дата публикации, кнопка `Открыть`).
- Экспорт: не требуется.

#### 5.3.4 Стейкхолдер: статистика на страницах запросов/портфеля/идеи

Стейкхолдерская статистика показывает:

- активность по его запросам (что происходит и как быстро двигается);
- прогресс работ и риски по рабочим областям одобренных идей (по задачам Bitrix24).

**A) `ЛК стейкхолдера → Мои запросы` (активность авторов по запросам)**

- Блок `Активность по моим запросам`:
  - `Новых идей (за 7 дней)` — сколько идей подано по моим запросам за 7 дней.
  - `Активных авторов (за 7 дней)` — сколько уникальных авторов подало идеи по моим запросам за 7 дней.
  - `Решений принято (за 7 дней)` — сколько решений (`Одобрена`/`Отклонена`) принято за 7 дней по моим запросам.
  - `Идей на решение (сейчас)` — сколько идей сейчас в статусе `На рассмотрении` по моим запросам (текущая очередь).
- В таблице запросов (в строке запроса): показывать `Идей: N` и бейдж `+N`, если появились новые идеи/обновления.

**B) `ЛК стейкхолдера → Портфель идей` (прогресс и риски)**

- Блок `Прогресс и риски по одобренным идеям` (по задачам Bitrix24):
  - `Идей в реализации` — сколько идей в статусе `Одобрена` по направлениям стейкхолдера.
  - `Идей с рисками` — сколько одобренных идей имеют хотя бы одну просроченную задачу.
  - `Задач на 7 дней` — сколько **открытых** задач по одобренным идеям имеют дедлайн в ближайшие 7 дней.
    - Как считаем: открытые задачи Bitrix24 по одобренным идеям, у которых срок наступает в ближайшие 7 дней (включая сегодня).
    - Примечание: просроченные задачи (срок уже прошёл) не входят в показатель и учитываются через метрику `Идей с рисками`.
- В списке одобренных идей: показывать только пометку риска (например, `Есть просроченные задачи`), а детализацию прогресса и список рисков — внутри страницы идеи (см. п. C).

**C) Страница идеи в ЛК стейкхолдера (прогресс внутри идеи)**

- Внутри страницы идеи показывать блок `Работы и риски`:
  - прогресс задач (как в п. B);
  - список `Топ-5 рисков` (задачи с просрочкой или задачами с дедлайном в ближайшие 7 дней): название, дедлайн, ссылка Bitrix24.

**D) `ЛК стейкхолдера → Админ. панель` (сквозная статистика по личным задачам)**

- Блок `Мои задачи в Bitrix24`:
  - `Активных задач`, `Просроченных`, `На 7 дней` (дедлайн в ближайшие 7 дней);
  - таблица задач (название, дедлайн, связанная идея/проект, кнопка `Открыть`).

- Экспорт: доступен на страницах `Мои запросы`, `Портфель идей`, `Админ. панель` (Excel).

#### 5.3.5 Координатор: операционная статистика в CRM (Bitrix24)

- Принцип: координаторская операционная статистика строится **в Bitrix24** (смарт‑процессы и задачи); портал не дублирует управление и показывает только ссылки на нужные представления.
- Экран: `CRM Bitrix24 → Дашборд координатора` (единый экран).

**A) График «Воронка обработки идей (сейчас)» — заменяет отдельные метрики очередей**

График показывает **одновременно** все ключевые очереди по идеям и заменяет отдельные счётчики вида «Очередь модерации», «Идеи на решение» и т.п.

- Срез: состояние **на текущий момент** (сколько идей находится в каждом статусе прямо сейчас).
- Фильтры графика: `Направление` (`Все`/одно), `Стейкхолдер` (`Все`/один).
- Шаги воронки (каждый шаг — отдельная метрика):
  - `На модерации (сейчас)`
    - Что показывает: сколько идей ожидают первичной проверки координатором.
    - Как считаем: количество идей со статусом `На модерации`.
    - Источник: смарт‑процесс «Идея» (Bitrix24) → статус.
  - `На доработке (сейчас)`
    - Что показывает: сколько идей возвращены автору на исправление и «стоят» у автора.
    - Как считаем: количество идей со статусом `На доработке`.
    - Источник: смарт‑процесс «Идея» (Bitrix24) → статус.
  - `На рассмотрении (сейчас)`
    - Что показывает: сколько идей ожидают решения стейкхолдера.
    - Как считаем: количество идей со статусом `На рассмотрении`.
    - Источник: смарт‑процесс «Идея» (Bitrix24) → статус.
  - `В реализации (Одобрена, сейчас)`
    - Что показывает: сколько идей одобрено и находится в стадии выполнения (есть рабочая область/задачи).
    - Как считаем: количество идей со статусом `Одобрена` (исключая `Архив`).
    - Источник: смарт‑процесс «Идея» + связанные проекты/задачи (Bitrix24).

**B) Метрики рисковых идей (сейчас)**

Блок нужен, чтобы координатор быстро видел «где горит» в реализации и мог подключить стейкхолдера/команду.

- `Проектов с риском (сейчас)`
  - Что показывает: сколько одобренных идей имеют просроченные задачи.
  - Как считаем: количество идей со статусом `Одобрена`, у которых есть **хотя бы одна** открытая задача Bitrix24 со сроком в прошлом.
  - Источник: задачи Bitrix24, привязанные к рабочей области одобренной идеи.

- `Просроченных задач в проектах (сейчас)`
  - Что показывает: объём просрочки в работах по одобренным идеям.
  - Как считаем: количество **открытых** задач Bitrix24 по одобренным идеям, у которых срок уже прошёл.
  - Источник: Bitrix24 задачи.

- `Задач на 7 дней в проектах (сейчас)`
  - Что показывает: объём работ, срок по которым наступает в ближайшую неделю (прогноз загрузки/рисков).
  - Как считаем: количество **открытых** задач Bitrix24 по одобренным идеям, у которых срок наступает в ближайшие 7 дней (включая сегодня).
  - Источник: Bitrix24 задачи.

**C) Метрики задач координатора (административных, сейчас)**

Блок нужен для контроля регламентов и операционной нагрузки координатора.

- `Активных задач координатора (сейчас)`
  - Что показывает: сколько задач назначено на координатора и ещё не завершено.
  - Как считаем: количество задач Bitrix24, где координатор — ответственный, и статус задачи не `Завершена`.
  - Источник: Bitrix24 задачи.

- `Просроченных задач координатора (сейчас)`
  - Что показывает: сколько задач координатора имеют срок в прошлом.
  - Как считаем: количество активных задач координатора, у которых срок уже прошёл.
  - Источник: Bitrix24 задачи.

- `Задач координатора на 7 дней (сейчас)`
  - Что показывает: сколько задач координатора «на горизонте недели».
  - Как считаем: количество активных задач координатора, у которых срок наступает в ближайшие 7 дней (включая сегодня).
  - Источник: Bitrix24 задачи.

#### 5.3.6 Руководство: критичные метрики платформы (минимальный набор)

- Доступ: роль `Руководство` (и при необходимости координатор) видит сводный экран (только просмотр) + `Экспорт в Excel`.
- Экран: `Админ/Аналитика → Сводка`.
- Фильтры: `Период`, `Направление` (`Все`/одно), `Стейкхолдер` (`Все`/один).
- На экране всегда отображается `Данные обновлены: ДД.ММ.ГГГГ HH:MM` (последнее успешное обновление из Bitrix24).

**A) Блок «Сейчас» (оперативная картина)**

- `Активных запросов (сейчас)`
  - Что показывает: сколько запросов сейчас открыто для сбора идей.
  - Как считаем: количество запросов со статусом `На сборе идей` (исключая `Закрыт`).
  - Источник: портал (статус запроса синхронизирован с Bitrix24).

- `Идей на решение (сейчас)`
  - Что показывает: текущая «очередь решений» стейкхолдеров.
  - Как считаем: количество идей со статусом `На рассмотрении` (все направления/или по выбранным фильтрам).
  - Источник: портал (статус идеи синхронизирован с Bitrix24).
  - Примечание: показатель не зависит от выбранного `Периода` — это состояние на текущий момент.

- `Проектов в реализации (сейчас)`
  - Что показывает: сколько проектов сейчас находится в стадии выполнения.
  - Как считаем: количество идей со статусом `Одобрена` (исключая `Архив`).
  - Источник: портал (статус идеи) + Bitrix24 (наличие рабочей области проекта).
  - Примечание: если проект закрыт (идея переведена в `Архив`), он не считается «в реализации».

- `Проектов с риском (сейчас)`
  - Что показывает: сколько проектов требуют внимания из‑за просроченных работ.
  - Как считаем: количество проектов в реализации, у которых есть **хотя бы одна** открытая задача Bitrix24 со сроком в прошлом (`дедлайн < сегодня`).
  - Источник: Bitrix24 (задачи проекта).
  - Примечание: учитываются только задачи, относящиеся к рабочей области одобренной идеи (связь «идея ↔ проект/задачи» ведётся в Bitrix24).

**B) Блок «За период» (динамика и результат)**

- `Новых запросов (за период)`
  - Что показывает: активность стейкхолдеров по постановке задач.
  - Как считаем: количество запросов, опубликованных в выбранный период (по дате публикации).
  - Источник: портал.

- `Подано идей (за период)`
  - Что показывает: участие и приток предложений.
  - Как считаем: количество идей, созданных в выбранный период (по дате подачи/создания) **кроме** статуса `Черновик`.
  - Источник: портал.

- `Решений принято (за период)`
  - Что показывает: скорость прохождения «точки решения».
  - Как считаем: количество идей, которые в выбранный период были переведены в один из финальных статусов решения: `Одобрена` или `Отклонена` (по дате смены статуса).
  - Источник: Bitrix24 (журнал смены статусов) → портал.

- `Доля одобрения (за период)`
  - Что показывает: «качество» входящего потока идей и баланс решений.
  - Как считаем: `Одобрено за период / (Одобрено за период + Отклонено за период) * 100%`.
  - Источник: Bitrix24 → портал.
  - Примечание: если в периоде `Решений принято = 0`, доля одобрения не рассчитывается и показывается как `—`.

- `Архивировано проектов (за период)`
  - Что показывает: сколько проектов завершено внутри программы.
  - Как считаем: количество идей, переведённых в статус `Архив` в выбранный период (по дате архивации).
  - Источник: Bitrix24 → портал.

- `Опубликовано кейсов (за период)`
  - Что показывает: сколько результатов стало публичными.
  - Как считаем: количество кейсов, **впервые** переведённых в статус публикации `Опубликовано` в выбранный период.
  - Источник: портал (статус публикации кейса, синхронизирован с Bitrix24).
  - Примечание: повторная публикация ранее опубликованного кейса не увеличивает показатель за период.

**C) Блок «Воронка результатов» (за период)**

- Показывать цепочку и конверсии: `Подано идей` → `Решений принято` → `Одобрено` → `Архивировано` → `Опубликовано кейсом`.
- Для каждого перехода отображать процент конверсии (например, `Решений принято / Подано идей`).
  - Определения шагов:
    - `Подано идей` — как в блоке B (без `Черновик`, по дате подачи в период).
    - `Решений принято` — как в блоке B (перевод в `Одобрена` или `Отклонена` в период).
    - `Одобрено` — количество идей, переведённых в статус `Одобрена` в выбранный период (по дате смены статуса).
    - `Архивировано` — как в блоке B (перевод в `Архив` в период).
    - `Опубликовано кейсом` — как в блоке B (впервые `Опубликовано` в период).
  - Правило расчёта конверсии: `Шаг N / Шаг N-1 * 100%`; если знаменатель равен 0 — показывать `—`.

- Экспорт: `Экспорт в Excel` по текущим фильтрам (карточки + воронка), без CSV/PDF.

### 5.4 Интеграции и надёжность

#### 5.4.1 Кэширование данных Bitrix24 (обязательно)

- Портал хранит у себя копию данных, которые нужны для портфеля проектов и статистики, чтобы страницы открывались быстро и не «падали» при временных сбоях Bitrix24.
- Что именно кэшируем:
  - справочники (направления, перечень запросов и идей, статусы, связи «идея ↔ проект/задачи»);
  - состав команд по проектам (кто состоит в команде и с какой ролью);
  - данные для контроля сроков в проектах (сколько задач открыто, сколько просрочено, какие сроки ближайшие).
- Как обновляем:
  - оперативные данные (статусы, сроки задач, состав команд) — каждые 30 минут;
  - справочники — минимум 1 раз в сутки, а также по ручной команде координатора «Обновить данные».

#### 5.4.2 Почтовые уведомления

- Письма отправляются только по событиям: подтверждение регистрации, напоминания, письмо с материалами после события.

### 5.6 Архивация идей (преобразование в проекты)

- Архивация — это «закрыть проект внутри программы»: перевести идею/проект в статус `Архив`.
- Инициатор: стейкхолдер (только по своим направлениям). Условие: идея в статусе `Одобрена`.
- При архивации стейкхолдер обязан заполнить `Сопроводительное сообщение` (видно команде и координатору): краткий итог, что получилось, где смотреть материалы/демо; если проект закрыт без результата — указать причину.
- Что происходит при архивации:
  - в Bitrix24 закрывается рабочее пространство проекта (группа/проект переводится в режим завершения/архива; новые задачи и процессы по проекту больше не запускаются);
  - портал фиксирует минимальную статистику на дату архивации (для аналитики и истории проекта): длительность (от одобрения до архивации), состав команды (количество участников и роли), задачи Bitrix24 (всего/закрыто/открыто, из открытых — просрочено), количество материалов и сколько из них помечены как публичные;
  - автоматически создаётся кейс со статусом публикации `Черновик` (см. 5.1.1). По умолчанию он не отображается в публичном портфеле проектов.
- Отображение:
  - в кабинете стейкхолдера/координатора архивная идея/проект остаётся доступной для просмотра вместе с сопроводительным сообщением, ссылками на материалы и ссылкой на Bitrix24;
  - в публичном портфеле проектов архивные идеи/проекты не показываются — публично отображаются только кейсы со статусом публикации `Опубликовано`.

## 6. UX/Интерфейс

- Базируется на принципах минималистичных карточек из Ф2: одна главная кнопка «Подробнее/Перейти», без индикаторов заполненности команды, только текущий состав и датированные бейджи («Опубликовано/Обновлено ДД.ММ.ГГГГ»; для событий — статусы `Завершено/Отменено`, а не «Просрочено»).
- Портфель идей стейкхолдера: карточки (одобренные/архивные) + блок фильтр-чипов, быстрые ссылки на материалы/Bitrix24.
- События: лёгкая форма регистрации (ФИО, e-mail, роль), подтверждение на экране + уведомление.

## 7. Данные и интеграции

- Новые сущности:
  - Кейс: ссылка на исходную архивную идею/проект, название, направление, описание (проблема/цель/решение/итог), показатели результата, материалы (ссылки с признаком публичности), команда (берётся из Bitrix24), статус публикации (`Черновик`/`Опубликовано`/`Снято с публикации`), дата публикации, дата снятия с публикации, причина снятия (обязательна при снятии).
  - Событие: название, направление, формат (онлайн/офлайн), дата‑время начала и окончания, место или ссылка, спикеры, программа/описание, материалы, статус (`Черновик`/`Опубликовано`/`Завершено`/`Отменено`/`Архив`), даты создания/публикации.
  - Регистрация на событие: событие, пользователь (если авторизован), ФИО, e‑mail, роль участника, дата‑время регистрации, статус регистрации (`Активна`/`Отменена пользователем`/`Отменена организатором`).
- Расширения к идее/проекту: дата архивации и сопроводительное сообщение стейкхолдера; связь «идея/проект → кейс» (кейс‑черновик создаётся автоматически при архивации); ссылки на материалы (с признаком публичности).
- Материалы: файлы остаются в Bitrix24 или во внешнем хранилище; на портале храним только ссылки.

## 8. Приёмочные критерии

1. Публичный каталог кейсов доступен без авторизации: в каталоге отображаются только кейсы со статусом публикации `Опубликовано`, они открываются в карточку «только просмотр»; карточки показывают дату «Опубликовано/Обновлено», без кнопок управления командой/решений.
2. События: календарь/список, регистрация, напоминания, материалы после события работают; регистрация гостя фиксируется, есть кнопка «Добавить в календарь» и ссылка на трансляцию или адрес.
3. Статистика реализована минимальным набором (п. 5.3): публичный блок «За 7 дней» на странице «Банк инноваций», блок «Мой вклад» у участника, встроенные блоки у стейкхолдера и сводка для руководства; все экраны — только просмотр, данные обновляются по расписанию, экспорт доступен только в Excel (без CSV/PDF); внешних подключений к системам аналитики нет.
4. Расширенный профиль (навыки/интересы, участие в проектах) доступен для просмотра/редактирования, поиск/фильтры по профилям работают.
5. При переводе идеи/проекта в `Архив` автоматически создаётся кейс со статусом публикации `Черновик`; опубликовать/снять с публикации может только стейкхолдер, при снятии причина обязательна; гостю доступны только кейсы со статусом публикации `Опубликовано`, публикация/снятие не меняет статус идеи и задачи в Bitrix24.

## 9. Зависимости и предпосылки

- Доступ к Bitrix24 для чтения данных о задачах, командах и материалах (чтение, без управления).
- Настроенная отправка e‑mail для писем по событиям.
- Внешние системы аналитики не требуются; используются внутренние блоки статистики и выгрузки Excel.
