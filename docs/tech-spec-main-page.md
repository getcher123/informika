# Техническое задание — новая Главная страница (публичная)

## 1. Цель
Сделать новую главную страницу платформы, которая:
- Быстро объясняет новичкам, что такое платформа и какие есть основные разделы.
- Продвигает ключевые активности (например, мероприятия или обучающие материалы) через рекламный баннер‑карусель.
- Показывает “самое важное прямо сейчас”: топ‑запросы на инновации и последние одобренные идеи (проекты), в которые можно вступить.

## 2. Роли и доступ
- **Гость / неавторизованный пользователь**: видит все блоки страницы, но действия “участия” должны вести к авторизации (если действие требует входа).
- **Авторизованный пользователь**: видит страницу так же, но может переходить в детали и выполнять доступные действия (например, подать заявку в команду проекта).

## 3. Размещение и навигация
- В продукте: URL **`/`** (главная).
- В мокапах: можно реализовать как обновление `Layouts/Mockups/landing.html` (или новый файл, если нужно сохранить старую версию).
- Логотип в хедере ведёт на главную.
- Верхнее меню (сквозное): `Банк инноваций`, `Кейсы`, `Календарь мероприятий`, `Конкурсы`, заглушки `Медиасентр`, `Обучение`, `Войти/Аватар`.

## 4. Структура страницы (сверху вниз)

### 4.1 Рекламный баннер (карусель) с пагинацией
**Назначение:** промо важной активности (по умолчанию — мероприятия; в будущем также курсы/обучение).

**Состав слайда:**
- Изображение (адаптивное, без “обрезки смысловой части” на мобайле).
- Короткий тип (например: `Мероприятие` / `Курс`).
- Заголовок (1–2 строки).
- Подзаголовок/описание (1–2 строки).
- Дата/время (если применимо).
- CTA‑кнопка: `Подробнее` (ведёт в карточку события/курса).
  - Для событий: `Layouts/Mockups/event-detail.html`.

**Навигация карусели:**
- Пагинатор‑точки (количество = количеству слайдов).
- Кнопки `←` / `→` (при наличии ≥ 2 слайдов).
- Переключение свайпом на мобильных.
- Автопрокрутка: **включена**, интервал 6–8 секунд.
- Автопрокрутка **останавливается** при:
  - наведении мыши на карусель,
  - фокусе внутри карусели (Tab‑навигация),
  - открытом меню/модальном окне (если используется).

**Правила наполнения слайдов (без двусмысленности):**
- Источник — **события** (раздел “Календарь мероприятий”) с отметкой “Показывать на главной”.
- Если таких событий **0**: карусель не показывать, следующий блок поднимается вверх.
- Если событий **1**: показывать один слайд, скрыть стрелки и пагинатор.
- Если событий **2+**: показывать стрелки и пагинатор.
- Порядок: по полю “Порядок показа на главной” (1…999), затем по ближайшей дате события.

### 4.2 Текстовый блок “О платформе” (для новых пользователей)
**Назначение:** коротко объяснить смысл терминов и логику участия.

**Содержимое (текст в 2–4 предложения):**
- Что такое **Запрос**: потребность компании, к которой участники предлагают решения.
- Что такое **Идея**: предложение участника по конкретному запросу.
- Что такое **Проект**: одобренная идея, по которой формируется команда и ведётся работа.
- Призыв: “перейти в Банк инноваций и выбрать запрос/проект”.

**Вёрстка:**
- Текст занимает всю ширину контентной области (не в колонках).
- Внутри текста допустимы 1–2 ссылки: `Банк инноваций`, `Календарь мероприятий`.

### 4.3 Блок хайлайтов №1: “Важные запросы сейчас” (3 карточки, 3 колонки)
**Назначение:** подсветить 3 наиболее приоритетных запроса, по которым нужно больше идей.

**UI‑компонент (без придумывания нового вида):**
- Использовать **тот же вид карточки**, что и в `Layouts/Mockups/requests.html` (карточка запроса в гриде, классы `ideas-grid` / `idea-card`).

**Правила выборки:**
- Берём только запросы со статусом `На сборе идей`.
- Основной источник: запросы с отметкой “Показывать на главной”.
- Если таких запросов меньше 3 — добираем до 3 из остальных активных запросов:
  - сначала с ближайшим дедлайном,
  - затем по дате публикации (новее выше).
- Порядок: “Порядок показа на главной”, затем правила добора.

**Карточка запроса (каждая из 3):**
- Структура и элементы — **как в карточке на странице “Банк инноваций”** (`Layouts/Mockups/requests.html`):
  - Бейдж направления (цветной).
  - Бейдж даты `Сбор идей до <ДД.ММ>` (как в макете).
  - Бейдж `+N` (опционально, если есть непросмотренные события по запросу).
  - Название (кликабельно).
  - Описание (2 строки + многоточие).
  - В футере: индикатор статуса `На сборе идей` + кнопка `Подробнее`.
- Переход по `Подробнее`: `Layouts/Mockups/request-detail.html`.

### 4.4 Блок хайлайтов №2: “Новые проекты (одобренные идеи)” (3 карточки, 3 колонки)
**Назначение:** показать 3 последних проекта, где можно вступить в команду.

**UI‑компонент (без придумывания нового вида):**
- Использовать **тот же вид карточки идеи**, что и в `Layouts/Mockups/request-detail.html` в блоке “Поданные идеи” (классы `idea-submission`).

**Правила выборки:**
- Берём идеи со статусом `Одобрена` (в интерфейсе называем “Проект”).
- Сортировка: по дате одобрения (новее выше).
- Количество: 3.

**Карточка проекта:**
- Структура и элементы — **как в карточке идеи на странице запроса** (`Layouts/Mockups/request-detail.html`):
  - В шапке: индикатор статуса `Одобрено` (для этого блока всегда `status-indicator--approved`).
  - Рядом — бейдж даты (в этом блоке текстом показываем `Одобрено <ДД.ММ.ГГГГ>`; визуально — как `idea-submission__date`).
  - Название (кликабельно).
  - Описание (2 строки + многоточие).
  - В футере: блок автора (как в макете) + кнопка `Подробнее`.
- Переход по `Подробнее`: `Layouts/Mockups/idea-detail.html` (на этой странице участник может подать заявку на вступление в команду).
- Сценарий “вступить в команду” выполняется **внутри карточки идеи** (форма/процесс в мокапах: `Layouts/Mockups/join-team-form.html`).

### 4.5 Подвал (футер)
**Назначение:** стандартные сервисные ссылки и контакты.

**Минимальный состав:**
- Копирайт.
- Ссылка: `Контакты` (`Layouts/Mockups/contacts.html`).
- Ссылка: `Политика обработки ПДн` (`Layouts/Mockups/privacy-policy.html`).
- Ссылка: `Техническая поддержка` (`Layouts/Mockups/support-form.html`).

## 5. Состояния (обязательные)

### 5.1 Загрузка данных
- Пока данные баннера/запросов/проектов загружаются: показывать “скелетон” соответствующих карточек (без прыжков высоты).

### 5.2 Пустые данные
- Баннер: см. правила в 4.1.
- Запросы: если активных запросов нет — показывать нейтральный текст “Сейчас нет активных запросов. Загляните позже.”
- Проекты: если одобренных идей нет — показывать нейтральный текст “Пока нет новых проектов. Следите за обновлениями.”

### 5.3 Ошибка
Единое сообщение на месте блока:
- “Не удалось загрузить данные, попробуйте позже” + кнопка `Повторить`.

## 6. Адаптив и доступность (обязательно)
- Брейкпоинты: 320 / 768 / 1024 / 1440.
- Сетка карточек:
  - 1440: 3 колонки.
  - 1024: 2 колонки.
  - 768 и ниже: 1 колонка.
- Карусель:
  - управляется с клавиатуры (Tab/Enter),
  - элементы управления имеют `aria-label`,
  - фокус видимый.
- Все кликабельные элементы имеют достаточную “зону нажатия” на touch.

## 7. Критерии приёмки
- На странице есть 4 ключевых блока в правильном порядке: карусель → текст → топ‑запросы → новые проекты → футер.
- Карусель работает: стрелки/пагинатор/автопрокрутка/пауза на hover.
- В “Важных запросах” всегда показывается ровно 3 карточки (если в системе есть ≥ 3 активных запроса).
- В “Новых проектах” показывается до 3 карточек, сортировка соответствует дате одобрения.
- Все карточки ведут на соответствующие страницы (`Подробнее`), а действия “участия” у гостя переводят на авторизацию.
