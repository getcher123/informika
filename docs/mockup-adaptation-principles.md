# Принципы адаптации статичных макетов в Bitrix (по факту сверки бэкапа `www/`)

## Общие правила интеграции
- **Разделение статики и логики**: HTML/JS/CSS из мокапов кладём в шаблон `/local/templates/<название>/`, бизнес-логику — в компоненты/страницы `bitrix`/`local`. Не смешивать разметку с PHP-логикой.
- **Точки входа**: каждая страница подключает `bitrix/header.php`/`footer.php` и нужный шаблон. Без ядра или шаблона страница не работает — при локальной сборке нужны симлинки на рабочее `/bitrix` и `/local/templates`.
- **Маршруты и urlrewrite**: все публичные/ЛК страницы должны иметь стабильные URL, прописанные в `urlrewrite.php` и/или `.htaccess`; избегать завязки на GET-параметры вместо человекопонятных путей.
- **Статика**: макеты опираются на `/local/templates/in_studio_landing/assets/...`. При адаптации обязательно положить:
  - CSS: `base.css`, `layout.css`, `components.css`, page CSS (`landing.css`, `request-form.css`, `auth.css`, `profile.css`, `participant-cabinet.css`, `request-detail.css` и др.).
  - JS: `vendor.js`, `app.js`, `forms.js` (формы), `auth.js` (логин/регистрация).
  - IMG: логотипы/плейсхолдеры/баннеры (`logo-Inn.svg`, `user-default.svg`, `2002-*.webp`, `land-01.png` и т.п.).
- **Единый UI Kit/навигация**: добавляем внутреннюю страницу ui-kit и каталог ссылок (как `Layouts/Mockups/index.html`) внутри сайта для визуальной проверки соответствия макетам.
- **Мобильность/адаптив**: все шаблоны должны использовать брейкпоинты макетов (320/768/1024/1440) и BEM-классы; сверять mobile-first стили из мокапов.
- **Данные/CRM**: страницы, работающие с данными (например, `innovations/index.php`), должны использовать Bitrix API/CRM-entities; не «вшивать» моковые данные. При отсутствии модуля — выводить fallback/заглушку.

## По разделам (на основе фактической сверки)
- **Публичные страницы**: лендинг и деталки запросов/идей соответствуют структуре макетов, но требуют подключения всех ассетов и проверки блоков (CTA, карточки, фильтры, рейтинги).
- **Формы**: `form.php` и `join-team-form.php` реализуют многошаговые формы; нужно сверить с макетами drag&drop загрузку, валидацию, шаги и сообщения об ошибках. Отсутствует самостоятельная `idea-form` — требуется добавить.
- **Список запросов/идей**: реализован через CRM (`innovations/index.php`), нужно привести UI (фильтры, карточки, пустые состояния) к макетам `requests.html`.
- **Политики/статичное**: `privacy-policy.php` — сверить текст с макетом.
- **Личный кабинет участника**:
  - Есть страницы: `lk/index.php` (идеи), `applications.php`/`application_detail.php` (командные заявки), `notifications.php`, `profile.php`, `idea/add|edit.php`, `login.php`, `register.php`.
  - Сверить с макетами `participant-*`: табы, бейджи +N, статусы, фильтры, формы редактирования идеи, карточки заявок, профиль/уведомления.
- **Личный кабинет стейкхолдера**:
  - Есть страницы: `lk/stakeholder/index.php`, `portfolio.php`, `notifications.php`, `profile.php`, `request.php`/`request-detail.php`, `create-request.php`/`edit-request.php`, `idea.php`/`idea-edit.php`, `admin-board.php`.
  - Сверить с макетами `stakeholder-*`: таблицы запросов, карточки, метрики портфеля, панель (dashboard/tools), формы редактирования, уведомления.
- **Не реализовано**: `team-requests.html`, `stake-holder-dashboard.html` (отдельная страница), UI Kit/навигация, `idea-form.html` (отдельная страница подачи идеи вне ЛК).

## Практические шаги адаптации
1) Развернуть/подключить шаблон `/local/templates/in_studio_landing` целиком (CSS/JS/IMG).
2) Добавить ui-kit/навигацию для быстрого визуального контроля.
3) Прописать стабильные маршруты и urlrewrite под названия макетов.
4) Сверить каждый экран с макетом: блоки, фильтры, статусы, формы, сообщения, адаптив.
5) Для ЛК участника/стейкхолдера — привести структуры страниц и данные к макетам; добавить недостающие экраны (командные заявки, dashboard, idea-form).
6) Проверить зависимость от CRM/Bitrix API: настроить сущности/поля или предусмотреть заглушки.
7) Тест: прогнать все страницы с подключённым шаблоном, сравнить с макетами (desktop/mobile), исправить расхождения по UI/JS.
