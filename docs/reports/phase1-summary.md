# Отчёт по фазе 1 (MVP «Запросы и идеи»)

## Цели и достигнутые результаты
- Запущен базовый сценарий Ф1: «стейкхолдер публикует запрос → участники подают идеи → модерация → решение». Идея остаётся сущностью платформы до статуса «Архив»; после архивирования дальнейшее проектное сопровождение (ПК/НТК) происходит вне системы.
- Реализованы пользовательские роли Ф1 (стейкхолдер, участник, модератор, гость) с кастомным ЛК участника и CRM-интерфейсами Bitrix.
- Обеспечен единый словарь терминов, статусы идеи («Черновик» … «Одобрена» … «Архив»), согласованные с расширенным ТЗ.

## Реализованные задачи
1. Каталог запросов и идей (Банк инноваций): фильтры, карточки, статус-чипы, действия «Подать идею», «Сохранить черновик», «Отправить на модерацию».
2. Личный кабинет участника: вкладки «Мои идеи» и «Заявки в команду», фильтры по статусам, карточки с комментариями модератора/стейкхолдера.
3. Стейкхолдерский процесс: создание/публикация запросов, модерация, принятие решений (CRM Bitrix), фиксация статусов «На рассмотрении»/«Одобрена»/«Отклонена».
4. Навигация и интерфейсы: единый хедер, CTA, заглушки для неготовых разделов (ссылка на Банк инноваций).
5. Техническая синхронизация: описание сущностей и статусов в [tech-spec-F1.md](../specs/phases/tech-spec-F1.md) и [overview.md](../specs/overview.md), текущий отчёт.

## Релиз‑ноты
- **Роли:** доступ стейкхолдера, участника, модератора и гостя реализован в полном объёме Ф1.
- **Корневые сценарии:** публикация запросов, подача идей, модерация, уведомления, решения.
- **Интерфейсы:** витрина (Банк инноваций), карточки запросов/идей, ЛК участника, CRM Bitrix.
- **Терминология:** закреплены понятия «Запрос», «Идея», статус «Одобрена», статус «Архив» (переход к внешнему проекту).

## Рекомендации по фазе 2
1. Кастомный ЛК стейкхолдера с портфелем одобренных идей и ссылками на Bitrix24 (задачи/документы).
2. Интеграция с сервисом совместной работы: создание шаблонов задач и структуры документов при переводе идеи в «Одобрена».
3. Расширенные отчёты (статусы задач, активность команды) и уведомления для стейкхолдера/координатора.
4. Управление правами и аудит действий при совместной работе.
