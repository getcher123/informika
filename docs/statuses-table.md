# Статусы ключевых сущностей

| Сущность | Статусы (порядок/развилки) | Комментарии |
|----------|----------------------------|-------------|
| Запрос на инновацию | Черновик → На сборе идей → Закрыт | Черновик — подготовка и редактирование стейкхолдером; На сборе идей — опубликован, доступен для подачи идей; Закрыт — выставляется по дедлайну или вручную, скрывает кнопку подачи (источник: docs/tech-spec-F1.md, §4.1). |
| Идея | Черновик → На модерации → На доработке ↺ → На рассмотрении → (Одобрена → Архив) \| Отклонена | Черновик — личный черновик автора; На модерации — проверка координатором; На доработке — возврат с замечаниями (повторная отправка ограничена политикой до 3 попыток); На рассмотрении — решение стейкхолдера; Одобрена — запускает рабочее пространство Bitrix24; Архив — фиксация итога и передача во внешний проект; Отклонена — финальное решение с обоснованием (источники: docs/README.md, §3; docs/tech-spec-F1.md, §4.2; docs/informika-extended-tz.md, разделы 4 и F). |
| Заявка в команду | На рассмотрении → (Принята \| Отклонена) → Архив | На рассмотрении — появляется у автора идеи и координатора во «Входящих»; Принята — участник включён в блок «Моя команда», фиксируются роль и дата вступления; Отклонена — отображается в «Отправленных» с причиной; Архив — хранение завершённых/отклонённых заявок (источники: docs/informika-extended-tz.md, раздел 4.9; docs/automation-table.md). |
| Задача Bitrix24 (воркфлоу сопровождения) | Новая → В работе → На проверке → (Возврат ↺ \| Готово) | Воркфлоу применяется к задачам, созданным после статуса идеи «Одобрена»; Возврат переводит задачу обратно «В работе» с комментариями; флаг «Просрочено» используется как дополнительный индикатор, но не меняет статус (источник: docs/README.md, §5). |
