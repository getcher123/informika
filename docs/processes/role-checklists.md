# Чек-листы по ролям (Фазы 1–2)

## Стейкхолдер
- [ ] Создан и опубликован запрос (поля: проблема, критерии, сроки, вложения).
- [ ] Назначен трекер/координатор для запроса.
- [ ] Все поступившие идеи просмотрены, добавлены комментарии.
- [ ] Для идей, требующих корректировки, оформлены возвраты «На доработку» с обязательной фиксацией причины.
- [ ] Для каждой идеи принято решение: «Одобрена», «Отклонена», «На доработке».
- [ ] Для одобренных идей: проверен состав команды, подтверждён старт работы в Bitrix24.
- [ ] Перед выводом на ПК/НТК: актуализирован пакет документов в папке `/ПК-НТК`, заполнён чек-лист готовности.

## Координатор
- [ ] Назначены модераторы и проверены права доступа.
- [ ] Отслеживаются идеи в статусах «На модерации»/«На доработке» (срок ≤ 5 рабочих дней).
- [ ] При статусе «Одобрена» автоматически создана рабочая область Bitrix24 (канбан, папки, шаблоны задач).
- [ ] Проверены обязательные роли команды; при пустых слотах отправлены уведомления.
- [ ] Мониторинг просроченных задач и чек-листов; эскалация стейкхолдеру.
- [ ] Актуализирован раздел `/Отчёты` (еженедельный экспорт активности).

## Автор идеи / Участник
- [ ] Черновик идеи заполнен полностью (заголовок, описание, ожидаемый эффект, вложения).
- [ ] При возврате на доработку внесены правки и повторная отправка выполнена в срок.
- [ ] Заявки в команду обработаны (ответы на приглашения, комментарии).
- [ ] После одобрения идеи: принят приглашение в Bitrix24, изучен `README-по-команде.md`.
- [ ] Выполняются назначенные задачи и чек-листы, комментарии оставляются в канбане.
- [ ] Материалы по ПК/НТК загружены в соответствующие папки.

## Модератор
- [ ] Проверены новые запросы/идеи на полноту и корректность терминологии.
- [ ] Замечания и причины возврата документированы в истории.
- [ ] Сроки доработки контролируются (напоминания авто/ручные).
- [ ] Перед переводом в «Одобрена» подтверждена корректность данных и отсутствие дубликатов.
- [ ] Журнал переходов статусов экспортирован при необходимости (CSV).

## Эксперт / Ментор / Трекер
- [ ] Получены приглашения в Bitrix24 и подтверждены роли.
- [ ] Оставлены оценки/комментарии по идеям в отведённые сроки.
- [ ] Контроль выполнения задач своей зоны ответственности (канбан, чек-листы).
- [ ] Фиксируются риски и блокеры (в карточке задачи или в комментариях к идее).
- [ ] Подготовлены рекомендации для ПК/НТК (при участии в финальной проверке).

## Руководство / PMO
- [ ] Получен консолидированный отчёт (еженедельно/ежемесячно).
- [ ] Проверены KPI по направлениям (количество идей, процент готовности).
- [ ] Замечания/решения переданы координаторам и стейкхолдерам.
