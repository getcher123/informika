# Описание интерфейсов (структурированный формат)

## 1. Главная страница (публичная)

### 1.1 Верхний блок
- Элементы:
  - Хедер с пунктами «Банк инноваций», «Конкурсы», заглушками «Медиасентр» и «Обучение».
  - Кнопка входа/аватар пользователя.

### 1.2 Промо-блоки
- «Как принять участие?»:
  - Карточка «Стать участником» (CTA → регистрация).
  - Карточка «Подать заявку».
- Промо-блок «Стать участником» (подробный переход к регистрации).

## 2. Страница «Банк инноваций»

### 2.1 Панель фильтров
- Элементы:
  - Dropdown «Статус запроса».
  - Dropdown «Направление».
- Поле «Автор» (input с автодополнением).
  - Строка поиска.
  - Переключатель сортировки (по дедлайну/по дате создания).

### 2.2 Грид карточек запросов
- Элементы каждой карточки:
  - Направление (одно из: Строительство, Эксплуатация, Безопасность, Цифровизация, Клиентский сервис, Другое).
  - Статус-чип (Активен = «На сборе идей», Закрыт).
  - Краткое описание.
  - Дедлайн.
  - Счётчик идей (цифра показывает общее количество поступивших идей; если есть новые непросмотренные — значок подсвечивается).
  - Кнопка «Подробнее» (ведёт на страницу запроса).

## 3. Страница запроса

### 3.1 Шапка
- Элементы:
  - Название запроса.
  - Направление (Строительство / Эксплуатация / Безопасность / Цифровизация / Клиентский сервис / Другое).
  - Статус: «На сборе идей» (активен) или «Закрыт».
  - Дедлайн.
  - Краткое описание проблемы.
  - Список вложений стейкхолдера.

### 3.2 Грид идей
- Элементы:
  - Карточки идей (см. раздел 4) в виде сетки.
  - Индикаторы новых действий:
    - Иконка «• Новые» появляется в правом верхнем углу карточки, если координатор или стейкхолдер ещё не просмотрели свежие комментарии или решения.
    - При наведении всплывает подсказка «Есть непросмотренные обновления».
    - После открытия карточки идеи и просмотра вкладок «Описание»/«Комментарии» индикатор автоматически исчезает.

### 3.3 Панель действий стейкхолдера
- Элементы:
  - Кнопка **«Редактировать запрос»**.
  - Кнопка **«Закрыть запрос»**.

### 3.4 Модальное окно «Отправить на доработку»
- Триггер: кнопка «Отправить на доработку» в карточке идеи (см. раздел 5.3).
- Поля:
  - «Причина возврата» (обязательное текстовое поле).
  - Кнопки «Отправить» и «Отмена» (уведомление автору отправляется автоматически).

### 3.5 Кнопка «Подать идею» и страница создания
- Кнопка «Подать идею» расположена:
  - В правой части шапки над гридом идей (видна только авторизованным участникам).
  - В карточке запроса внутри личного кабинета участника.
- Состояния:
  - Активно, когда статус запроса — «На сборе идей».
  - Не отображается или отключено, когда запрос переведён в «Закрыт».
- Нажатие переводит пользователя на отдельную страницу «Создание идеи» (без модальных окон).

#### 3.5.1 Страница «Создание идеи»

##### Шапка
- Хлебные крошки вида «Банк инноваций → <Название запроса> → Новая идея».
- Кнопка «Назад к запросу».

##### Основная форма
- Поля:
  - «Название идеи» (обязательное текстовое поле).
  - «Краткое описание/проблема».
  - «Ожидаемый эффект».
  - «Направление» (выбирается из списка, подтягивается из запроса, но доступно для уточнения при необходимости).
  - «Запрос» (read-only, отображает выбранный запрос).
  - Чекбокс согласия с правилами участия.

##### Блок вложений
- Drag-and-drop зона для файлов (pdf/doc/ppt) с отображением загруженных элементов.
- Поле для ссылок (например, на внешние прототипы).

##### Панель действий
- Кнопка «Сохранить черновик».
- Кнопка «Отправить на модерацию».
- Кнопка «Отмена» (возврат к странице запроса без сохранения).

##### Валидация и подсказки
- Inline-валидация обязательных полей (подсветка и текст подсказки).
- Баннер состояния (например, «Черновик сохранён» или «Отправлено на модерацию»).

## 4. Карточка идеи (в списках)

### 4.1 Поля карточки
- Направление (Строительство / Эксплуатация / Безопасность / Цифровизация / Клиентский сервис / Другое).
- Текущий статус идеи (Черновик, На модерации, На доработке, На рассмотрении, Одобрена, Отклонена, Архив).
- Автор.
- Краткое описание.
- Дата обновления.
- Кнопка «Подробнее» (переход на страницу идеи, см. раздел 5).

### 4.2 Сценарии отображения
- **Автор (в «Мои идеи»):**
  - Бейдж статуса модерации (цветовая схема: серый — Черновик, жёлтый — На модерации, красный — На доработке, синий — На рассмотрении, зелёный — Одобрена, тёмно-серый — Архив).
  - Индикатор новых комментариев:
    - Счётчик иконкой чата показывает непрочитанные сообщения.
    - Расположен в правом верхнем углу карточки.
    - Сбрасывается после открытия страницы идеи и просмотра вкладки «Комментарии».
  - Кнопка «Редактировать» (доступна до статуса «На рассмотрении»).
  - Кнопка «Открыть карточку».
- **Стейкхолдер (портфель):**
  - Бейдж стадии:
    - «Одобрена» (базовый статус после решения).
    - «Готово к ПК» (подготовка к проектному комитету).
    - «Готово к НТК» (подготовка к научно-техническому комитету).
    - Цветовая кодировка: зелёный — «Одобрена», синий — «Готово к ПК», фиолетовый — «Готово к НТК».
  - Кнопки «Перейти к задачам», «Перейти к документам».
  - Ссылка на канбан конкретной идеи.
- **Участник (не автор, статус «Одобрена»):**
  - Кнопка «Вступить в команду».
  - Блок «Команда» с текущими участниками и закреплёнными ролями.

## 5. Страница идеи

### 5.1 Шапка
- Элементы:
  - Название идеи.
  - Направление (Строительство / Эксплуатация / Безопасность / Цифровизация / Клиентский сервис / Другое).
  - Текущий статус (Черновик, На модерации, На доработке, На рассмотрении, Одобрена, Отклонена, Архив).
  - Ссылка на запрос.
  - Бейдж стадии (для Одобрена: базовая / Готово к ПК / Готово к НТК).

### 5.2 Вкладки
- «Описание» — основной контент идеи.
- «Комментарии» — общение автора, координатора, стейкхолдера.
- «Вложения» — детализированный список файлов автора.

### 5.3 Блоки по ролям
- **Автор:**
  - Секция «Мои документы».
  - Кнопка «Редактировать идею» (если статус позволяет).
  - Индикатор обратной связи:
    - Плашка с текстом «Есть новое решение/комментарий» появляется, когда координатор или стейкхолдер оставили новые сообщения или сменили статус.
    - Плашка содержит ссылку на соответствующий раздел («Комментарии» или блок «Решение»).
    - Исчезает после просмотра обновлённого раздела.
  - Кнопка «Открыть задачи в Bitrix24» (когда статус «Одобрена»).
- **Участник (не автор):**
  - Кнопка «Вступить в команду» (открывает модальное окно, см. 5.4).
  - Блок «Команда» с ролями и текущими участниками.
- **Стейкхолдер:**
  - Блок «Решение» с кнопками «Одобрить», «Отклонить», «Отправить на доработку».
  - Кнопки «Перейти к задачам» и «Перейти к документам» (активны в статусе «Одобрена»).

### 5.4 Модальное окно «Вступить в команду»
- Поля:
  - Dropdown «Целевая роль» (Тимлид, Аналитик, Разработчик, Тестировщик, Коммуникационный специалист, Эксперт/Ментор, Дизайнер, DevOps/Инженер внедрения).
  - Мультиселект «Уровень компетенции» (Junior, Middle, Senior, Expert).
  - Textarea «Опыт/успехи».
  - Загрузка портфолио/резюме (опционально).
  - Чекбокс согласия с правилами (контакты подтягиваются автоматически).
  - Кнопки «Отправить заявку» и «Отмена».

## 6. Личный кабинет участника

### 6.1 Меню
- Вкладки: «Мои идеи», «Заявки в команду», «Уведомления», «Профиль».

### 6.2 Вкладка «Мои идеи»
- Элементы:
  - Фильтры по статусам (Черновик, На модерации, На доработке, На рассмотрении, Одобрена, Отклонена, Архив).
  - Список карточек идей (см. раздел 4) с быстрыми действиями.

### 6.3 Вкладка «Заявки в команду»
- Элементы:
  - Подвкладки: Все, Входящие, Отправленные, Моя команда, Архив.
  - Карточки заявок с кнопками «Принять», «Отклонить», «Подробнее» (кнопка открывает карточку заявки, см. 6.3.1).
  - Блок «Моя команда» (закрытые роли/участники).
  - Кнопка «Подать заявку» (ведёт на страницу идеи и открывает модальное окно, см. 5.4).

#### 6.3.1 Карточка заявки (панель/модальное окно)
- Поля:
  - Статус заявки (На рассмотрении, Принята, Отклонена, Архив).
  - Целевая роль.
  - Уровень компетенции.
  - Поле «Опыт/успехи».
  - Прикреплённое портфолио/резюме (если загружено).
  - Контактные данные из профиля (только для просмотра).
- Действия:
  - Кнопка «Принять» (назначение роли и даты вступления).
  - Кнопка «Отклонить» (с обязательным комментарием).
  - Кнопка «Запросить уточнение» (отправка вопроса заявителю).

### 6.4 Вкладка «Уведомления»
- Лента событий: решения модератора и стейкхолдера, ответы по заявкам, обновления задач/документов; фильтры по типу события; быстрые ссылки на карточки.

### 6.5 Профиль
- Поля:
  - Аватар.
  - ФИО.
  - Организация.
  - Роли.
  - Контакты.
  - Настройки уведомлений и смена пароля.
- Кнопка «Открыть задачи в Bitrix24».

## 7. Личный кабинет стейкхолдера

### 7.1 Вкладки
- «Мои запросы» — грид карточек запросов с фильтрами и индикатором «Новые идеи»:
  - В правом верхнем углу карточки отображается бейдж «+N», где N — количество новых идей или обновлений внутри запроса.
  - Бейдж исчезает после открытия карточки запроса или просмотра вложенных идей.
- «Портфель одобренных идей» — карточки по стадиям («Одобрена», «Готово к ПК», «Готово к НТК») с кнопками перехода к задачам/документам/канбану.
- «Панель управления» — кнопка «Открыть административную доску».
- «Уведомления» — лента входящих событий (решения, напоминания, заявки).

### 7.2 Карточка запроса внутри ЛК
- Элементы:
  - Блок деталей запроса (описание, сроки, вложения, статус «На сборе идей»/«Закрыт»).
  - Грид идей (карточки с статусами и стадиями).
  - Боковая панель «Примечания координатора».

### 7.3 Вкладка «Панель управления»
- Единственный элемент: кнопка «Открыть административную доску» (канбан задач).

### 7.4 Вкладка «Уведомления»
- Лента событий роли (решения, напоминания, незаполненные роли, дедлайны ПК/НТК) с фильтрами и ссылками на карточки.

## 8. CRM модератора — пользовательские сценарии

- Список идей «На модерации» с фильтрами по направлению/автору.
- Карточка проверки: просмотр всех полей, вложений, комментариев.
- Кнопки: «Опубликовать», «Вернуть на доработку» (модальное окно причины), «Отклонить».
- Поле для комментария и фиксации решения (добавление события в журнал идеи).

## 9. CRM координатора

### 9.1 Страница «Команды идей»
- Таблица идей в статусе «Одобрена» с фильтрами по направлению, стадии, незаполненным обязательным ролям, активным заявкам.

### 9.2 Карточка идеи в CRM
- Вкладка «Команда» — список участников и ролей, отметки обязательных ролей.
- Вкладка «Кандидаты» — карточки претендентов с контактами, компетенциями и предыдущими участиями.
- Вкладка «Журнал» — список приглашений/решений координатора и комментариев к команде.

### 9.3 Действия
- «Пригласить в команду» (выбор пользователя/роли, отправка приглашения).
- «Принять/Отклонить/Запросить уточнение» по входящим заявкам.
- «Закрепить обязательную роль» (фиксирует заполненность ключевых ролей).
- Любое действие создаёт/обновляет карточку на административной доске.

## 10. Административная канбан-доска

### 10.1 Структура
- Колонки: Новая → В работе → Ожидает данных → На согласовании → Готово.
- Карточка содержит: заголовок, связанную сущность (запрос/идея/команда), чек-лист, дедлайн, исполнителей и комментарии.

### 10.2 Создание карточек
- Автоматически по триггерам (см. automation-table).
- Вручную координатором, стейкхолдером или модератором (для контроля процессов).
- Доступ ограничен ролями; координатор видит карточки стейкхолдера для эскалаций.

## 11. Разделы Bitrix24 (внешние)

### 11.1 Канбан задач
- Колонки: Новая, В работе, На проверке, Возврат, Готово.
- Поддержка чек-листов, исполнителей, наблюдателей, комментариев, файлов.

### 11.2 Диск идеи
- Структура папок:
  - `/Требования`
  - `/План`
  - `/Прототипы`
  - `/Тесты`
  - `/ПК-НТК`
  - `/Отчёты`
  - `/Коммуникации`
- Права доступа распределяются по ролям (автор/координатор — полный доступ, участники — свои папки, стейкхолдер — просмотр/утверждение).

### 11.3 Прямые переходы
- Ссылки на канбан задач и диск доступны:
  - В портфеле стейкхолдера (на карточках одобренных идей).
  - На странице идеи (блок ролей).
  - В личных кабинетах участников/стейкхолдера (кнопки в соответствующих вкладках).
