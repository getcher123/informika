# Расширенное описание интерфейсов Informika

Документ раскрывает состав экранов и элементов пользовательского интерфейса с детализацией по ролям, основан на `docs/interface-elements.md`, `docs/tech-spec-F2-full.md` и связанных ТЗ. Формулировки и подписи приведены в тех же названиях, что и в макетах и требованиях.

> Правила визуальных индикаторов и наборы значений:
>
> - Все статусы отображаются через круглый индикатор (цветная точка + подпись). Запросы: `На сборе идей`, `Закрыт`. Идеи: `Черновик`, `На модерации`, `На доработке`, `На рассмотрении`, `Одобрена`, `Отклонена`, `Архив`. Заявки: `На рассмотрении`, `Принята`, `Отклонена`, `Архив`.
> - Цветные бейджи применяются только для направления (`Строительство`, `Эксплуатация`, `Безопасность`, `Цифровизация`, `Клиентский сервис`, `Другое`), времени (`Сбор идей до ...`, `Дедлайн ...`). Бейджи срока всегда содержат конкретную дату (формат `ДД.ММ.ГГГГ`) или слово `Просрочено`.
> - Бейдж стадии идеи используется только для статуса `Одобрена`; готовность к ПК/НТК не визуализируется на витрине и отслеживается исключительно в Bitrix24. Ролевые бейджи заявок/команды: `Тимлид`, `Аналитик`, `Разработчик`, `Тестировщик`, `Коммуникационный специалист`, `Эксперт/Ментор`, `Дизайнер`, `DevOps/Инженер внедрения`.
> - Бейдж `+N` показывает количество непрочитанных событий по карточке (новые идеи, комментарии, документы, задачи) с момента последнего посещения экрана. После перехода внутрь карточки счётчик обнуляется автоматически.
> - Бейджи дедлайна отображают конкретную дату в формате `ДД.ММ.ГГГГ`; если срок просрочен, дата заменяется на слово `Просрочено`. Значение подтягивается из дедлайна запроса или последнего срока задачи Bitrix24 (для одобренных идей) и обновляется автоматически.

## 1. Главная страница

### 1.1 Публичная витрина (гость/неавторизованный пользователь)

- **Глобальный хедер**: логотип (клик по нему ведёт на главную), пункты меню `Банк инноваций`, `Конкурсы`, заглушки `Медиасентр`, `Обучение`, кнопка `Войти` (после авторизации заменяется на аватар). Заглушки открывают модальное сообщение «Раздел появится в следующей фазе».
- **Промо-блок «Как принять участие?»**: две карточки CTA `Стать участником` (кнопка ведёт на регистрацию) и `Подать заявку` (кнопка ведёт к каталогу запросов). У каждой карточки: заголовок, описание, кнопка.
- **Промо-блок «Стать участником»**: крупный баннер с описанием программы, списком преимуществ, кнопкой `Перейти к регистрации`.
- **Футер** (если используется в макете): копирайт, контакты, кнопка `Политика обработки ПДн`.

### 1.2 Авторизованный пользователь

- Хедер идентичен, но кнопка `Войти` заменяется на `Аватар + имя`. Клик открывает меню: `Личный кабинет`, `Выйти`.

## 2. Страница «Банк инноваций»

### 2.1 Блок статистики «За 7 дней» (виден всем ролям)

- Расположение: верх страницы `Банк инноваций`, **над фильтрами**.
- Формат: 4 карточки‑счётчика.
- Метрики:
  - `Новые запросы`
  - `Подано идей`
  - `Идей одобрено`
  - `Идей отклонено`
- Приватность: только агрегаты, без ФИО/почты/списков.

### 2.2 Общая панель фильтров (видна всем ролям)

- Dropdown `Статус запроса` (значения: Все, На сборе идей, Закрыт).
- Dropdown `Направление` (Строительство, Эксплуатация, Безопасность, Цифровизация, Клиентский сервис, Другое).
- Поле `Автор` с автодополнением по ФИО стейкхолдеров.
- Строка поиска «Введите название или ключевые слова».
- Переключатель сортировки (`По дедлайну`, `По дате публикации`).

### 2.3 Карточка запроса (грид)

- **Шапка**: бейдж направления (цветовой набор: `Строительство`, `Эксплуатация`, `Безопасность`, `Цифровизация`, `Клиентский сервис`, `Другое`), круглый индикатор статуса (`На сборе идей`, `Закрыт`), бейдж `+N`, если появились непросмотренные идеи.
- **Основной блок**: название (кликабельно и ведёт внутрь), 2 строки описания, подпись владельца (ФИО, подразделение).
- **Подвал**: пара `Сбор идей — <дата>`, счётчик `Идей: N`, кнопка `Подробнее`.

### 2.4 Поведение по ролям

- **Гость**: видит каталог, при попытке перейти к действиям («Подать идею») внутри карточки получает запрос авторизации.
- **Участник**: выполняет все действия (подача идеи, просмотр статусов) внутри раскрытой карточки; в плитке доступна только кнопка `Подробнее`.
- **Стейкхолдер (владелец)**: карточка помечается бейджем «Мой запрос», а кнопки `Редактировать/Закрыть` расположены внутри страницы запроса.
- **Модератор**: работает исключительно через подготовленную CRM Bitrix24;

## 3. Карточка запроса

### 3.1 Общая структура карточки запроса

- **Шапка**: название, направление (список направлений см. выше), индикатор статуса (`На сборе идей`, `Закрыт`), дедлайн (бейдж с датой `ДД.ММ.ГГГГ` или значением `Просрочено`), бейдж `+N`, блок внутренних кнопок по роли (`Редактировать`, `Закрыть`, `Подать идею`, действия модерации).
- **Инфоблок**: короткое описание, счётчик `Подано идей`, список вложений (название, тип, размер, иконка скачивания).
- **Секции страницы (расположены последовательно):**
  1. `Проблема/Цель`
  2. `Критерии принятия`
  3. `Сроки`
  4. `Материалы`
  5. `Комментарии` — единая лента сообщений (модератор, стейкхолдер, координатор, участник) с формой ответа
  6. `Переходы в Bitrix24` — кнопки `Перейти к задачам` и `Перейти к документам` (открываются в новом окне)
- **Блок «Идеи по запросу»**: грид компактных карточек (см. раздел 4) с локальными фильтрами «Все / На рассмотрении / На доработке / Одобрена / Отклонена». Все элементы в одной колонке; в плитке только `Подробнее` (решения принимаются внутри карточки идеи).
- **Компактная карточка идеи внутри запроса**:
  - Шапка: название идеи, дата публикации.
  - Средний блок: 1–2 строки описания и бейдж стадии (`На рассмотрении`, `Одобрена` и т.д.).
  - Нижний блок: кнопка `Подробнее`, мини-бейджи автора и даты обновления. Все дополнительные действия выполняются внутри раскрытой карточки.

### 3.2 Роль: гость / участник

- Кнопка `Подать идею` (активна только при статусе «На сборе идей»). Неавторизованный пользователь получает диалог «Авторизуйтесь, чтобы подать идею».
- Навигация осуществляется через хлебные крошки (`Банк инноваций → Запрос`); отдельная кнопка «Вернуться к каталогу» не используется.

### 3.3 Роль: стейкхолдер (владелец)

- Блок `Управление запросом`: кнопки `Редактировать запрос`, `Закрыть запрос`, бейдж `+N идей`.
- Решения по идее принимаются только из раскрытой карточки идеи (см. раздел 4). В списке идей доступны кнопки `Подробнее` и быстрые фильтры, но без действий `Одобрить/Отклонить`.

### 3.4 Роль: модератор

- В портале модератор не проводит действий; ему доступен просмотр и кнопка `Открыть в Bitrix24`. Комментарии и решения фиксируются через CRM Bitrix и отображаются в общей ленте `Комментарии`.

### 3.5 Модальное окно «Отправить на доработку» (стейкхолдер; модератор инициирует из Bitrix24)

- Поле `Причина возврата` (textarea, обязательное).
- Подвал: кнопки `Отправить`, `Отмена`, подсказка «Автор получит уведомление».

### 3.6 Страница «Создание идеи»

- Breadcrumb «Банк инноваций → <Запрос> → Новая идея», кнопка `Назад к запросу`.
- Форма:
  - `Название идеи*` (input, до 100 символов).
  - `Краткое описание*` (textarea до 1000 знаков).
  - `Ожидаемый эффект` (textarea).
  - `Файлы` (drag&drop, подпись «PDF/DOCX/XLSX до 10 МБ»).
  - `Ссылки` (input для URL, кнопка `Добавить ссылку`).
  - `Согласие на обработку ПДн*` (чекбокс).
- Кнопки формы: `Сохранить черновик`, `Отправить на модерацию`, `Отмена`.

### 3.7 Форма подачи запроса (страница `request-form.html`)

- Каркас: `form-container` с заголовком/подзаголовком и кнопкой возврата (`← Назад`). Под блоком — форма с двумя кнопками действий.
- Поля повторяют структуру HTML-макета:
  1. `Выберите направление*` — выпадающий список с placeholder `Выбор направления` и опциями: `IT-технологии`, `Строительство`, `Эксплуатация`, `Управление`. Используются кастомные select'ы (класс `form-select--decorated`).
  2. `Название запроса*` — текстовое поле с placeholder `Введите название запроса`.
  3. `Описание запроса*` — textarea, подсказка «Напишите основные задачи, которые решает ваш запрос».
  4. `Дедлайн приёма идей*` — поле даты (date-picker) с форматной подсказкой `дд.мм.гггг`; используется стандартный инпут с маской и валидацией на будущую дату.
  5. `Критерии приема идеи*` — textarea, подсказка «Перечислите критерии, по которым будет приниматься идея по запросу».
  6. `Файлы к запросу` — drag&drop-зона (`file-upload`): кнопка выбора + подпись «Выберите файлы или перетащите их сюда», поддерживается множественная загрузка.
- Кнопки внизу: `Сохранить черновик` (вторичная), `Опубликовать` (основная).

### 3.8 Страница редактирования запроса

- Тот же макет, что и `request-form.html`, с предзаполненными полями и заголовком «Редактирование запроса».
- Доступна владельцу запроса (стейкхолдер) в статусах `Черновик` и `На сборе идей`; поле `Направление` заблокировано при `На сборе идей`.
- Основной CTA: `Сохранить изменения`. Кнопка `Отмена` выполнена в виде текстовой ссылки, ведущей назад к карточке запроса.
- Опции `Удалить` и `Скопировать`/другие операции вынесены в меню `…` в шапке формы; `Удалить` открывает модалку подтверждения.
- Дополнительное действие: `Закрыть запрос` (вторичная кнопка или строка действия вверху). При нажатии — модалка подтверждения с текстом «Закрыть приём идей? Кнопка “Подать идею” перестанет быть доступна» и кнопками `Закрыть` / `Отмена`. Подтверждение переводит запрос в статус `Закрыт`, скрывает подачу идей и отображает уведомление.

## 4. Карточка идеи

### 4.1 Карточка идеи (унифицированная структура)

- **Шапка**: название, бейдж направления (цветной — `Строительство`, `Эксплуатация`, `Безопасность`, `Цифровизация`, `Клиентский сервис`, `Другое`), круглый индикатор статуса (значения: `Черновик`, `На модерации`, `На доработке`, `На рассмотрении`, `Одобрена`, `Отклонена`, `Архив`), кнопка `Открыть запрос`, внутренние кнопки по роли (решения стейкхолдера, редактирование автора и т.п.). В списках идей отображается только `Подробнее`; готовность к ПК/НТК фиксируется в Bitrix24 без отдельных бейджей и фильтров в карточке.
- **Секции страницы (расположены одной колонкой без вкладок):**
  1. `Описание` (текст, метрики, плашка `Мои документы` для автора).
  2. `Комментарии` (единая лента сообщений по идее с полем ввода; видна автору, стейкхолдеру, координатору, модератору и участникам команды).
  3. `Вложения` (таблица файлов).
  4. `Команда` (read-only состав ролей; для участника — кнопка `Вступить в команду`/плашка статуса участия; для остальных — только просмотр заполненности слотов, которые подтягиваются из Bitrix24).
  5. `Заявки в команду` (список входящих/отправленных заявок, отображается статус `На рассмотрении/Принята/Отклонена/Архив` без кнопок принятия/отказа; координатор обрабатывает заявки в Bitrix24; участник может `Отозвать` свою заявку при статусе «На рассмотрении»).
  6. `Решение стейкхолдера` (кнопки `Одобрить`, `Отправить на доработку`, `Отклонить`, отображение результата).
  7. `Задачи Bitrix24` (кнопка `Перейти к задачам`) и `Документы Bitrix24` (кнопка `Перейти к документам`).
- **Футер**: кнопки по ролям внутри портала ограничены автором (`Редактировать`), участником (`Вступить в команду`), стейкхолдером (`Одобрить/Отклонить/Доработка`). Для координатора отображаются только ссылки на Bitrix24/административную доску; действия выполняются во внешней CRM.
- **Вариации по ролям**:
  - Автор: плашка `Мои документы`, кнопка `Открыть задачи в Bitrix24` после одобрения.
  - Участник: кнопка `Вступить в команду`/плашка `Вы в команде`, доступ к блоку `Заявки в команду`.
  - Координатор: только просмотр состава и кнопка `Открыть в Bitrix24`.
  - Стейкхолдер: карточка решения + быстрые ссылки `Перейти к задачам/документам`.

### 4.9 Страница редактирования идеи

- Макет формы соответствует созданию идеи (раздел 3.6), но с заголовком «Редактирование идеи» и предзаполненными полями.
- Доступна автору в статусах `Черновик` и `На доработке`; в других статусах кнопки редактирования скрыты.
- Основной CTA: `Сохранить изменения`; рядом — текстовая ссылка `Отмена`. Опция `Удалить идею` вынесена в меню `…` в шапке формы.
- После сохранения пользователь возвращается в карточку идеи.

### 4.3 Роль: автор идеи

- Блок `Мои документы`: список загруженных файлов с кнопкой `Добавить файл`.
- Кнопка `Редактировать идею` (доступна в статусах Черновик, На доработке).
- Индикатор `Есть новое решение/комментарий` с ссылкой на соответствующий блок страницы.
- После одобрения — кнопка `Открыть задачи в Bitrix24`.

### 4.4 Роль: участник (не автор)

- Блок `Команда`: список ролей (иконка роли, ФИО участника, статус заполненности, данные из Bitrix24), без кнопок управления; управление ролями выполняется в Bitrix24.
- Кнопка `Вступить в команду` → открывает модальное окно (см. ниже).
- После решения по заявке в Bitrix24:
  - `На рассмотрении` — плашка `Заявка отправлена`, только просмотр.
  - `Принята` + метка `Ожидает подтверждения` — плашка + инструкция «Подтвердите приглашение из e‑mail (Bitrix24), иначе доступ не появится».
  - `Принята` + метка `Подтверждено` — плашка `Вы в команде (роль <роль>)` + кнопка `Открыть рабочее пространство` (Bitrix24) и быстрые ссылки `Перейти к задачам`/`Перейти к документам`.
- Доступ к блоку `Заявки в команду` (единая лента).

### 4.5 Роль: стейкхолдер

- Карточка `Решение`: кнопки `Одобрить`, `Отправить на доработку`, `Отклонить`. После выбора отображается результат (статус, комментарий) и кнопка `Изменить решение` (если доступно политикой).
- Кнопки `Перейти к задачам`, `Перейти к документам` (активны после «Одобрена»).

### 4.6 Роль: координатор

- В портале координатор видит только сводную информацию (состав команды, статусы заявок) и ссылки на Bitrix24/административную доску.
- Любые действия (`Назначить`, `Удалить`, работа с заявками) выполняются в CRM Bitrix; внутри карточки отображается кнопка `Открыть в Bitrix24`.

### 4.7 Роль: модератор

- Только просмотр: модератор использует Bitrix24 для решений; в карточке идеи отображаются метки изменений из Bitrix без дополнительных кнопок.

### 4.8 Модальное окно «Вступить в команду»

- Поля: dropdown `Целевая роль` (значения `Тимлид`, `Аналитик`, `Разработчик`, `Тестировщик`, `Коммуникационный специалист`, `Эксперт/Ментор`, `Дизайнер`, `DevOps/Инженер внедрения`), мультиселект `Уровень компетенции (Junior/Middle/Senior/Expert)`, textarea `Опыт/успехи`, загрузка портфолио, чекбокс `Согласен с правилами участия`.
- Кнопки: `Отправить заявку`, `Отмена`. Подсказка «Автор и координатор получат уведомление».

## 5. Модали и уведомления

- (Блок «Поделиться» не используется; распространение ссылки осуществляется стандартными средствами браузера).
- **Модальное окно «Запросить уточнение»** (координатор): поле вопроса, кнопки `Отправить`, `Отмена`.

## 6. Личный кабинет участника

### 6.1 Общие элементы

- Хедер `Личный кабинет`, иконка уведомлений, поиск по заявкам.
- Боковое меню: `Профиль`, `Мои идеи`, `Заявки в команду`, `Уведомления`.

### 6.2 Вкладка «Мои идеи» (автор)

- Над таблицей расположен блок `Мой вклад` (4 карточки):
  - `Идей предложено` (за всё время) + подпись `+N за 7 дней` (если есть).
  - `Идей одобрено` (за всё время).
  - `Проектов в реализации` (идеи участника в статусе `Одобрена`).
  - `Моё место среди активных участников` (за 30 дней): отображается как `#<место> из <N>` или `Не в рейтинге` (если за 30 дней не было подачи/доработки идей).
- Табличный вид идей автора:
  - Колонки: `Идея` (название), `Статус` (круглый индикатор), `Направление` (бейдж), `Дата обновления`, `Дедлайн` (бейдж срока), `+N` (непрочитанные). Действия: `Просмотр`, `Редактировать` (только статусы Черновик/На доработке).
  - Уведомления по идеям остаются карточками во вкладке «Уведомления» с кнопкой `Открыть/Перейти`.

### 6.3 Вкладка «Заявки в команду»

- Подвкладки (табы): `Все / Отправленные / Приглашения / Моя команда / Архив`.
- Важно: подтверждение участия для доступа Bitrix24 выполняется **через ссылку в e‑mail** (письмо отправляет Bitrix24 от имени модератора/стейкхолдера). В портале нет кнопок `Принять/Отклонить`.
- **Карточка заявки/приглашения (унифицированная)**:
  - Заголовок: `Идея: <название>` + ссылка `Открыть идею`.
  - Метаданные: тип `Заявка`/`Приглашение`, роль, инициатор (`Вы` или `Модератор/Стейкхолдер: <ФИО>`), дата обновления.
  - Статус: круглый индикатор TeamRequest (`На рассмотрении/Принята/Отклонена/Архив`).
  - Метка подтверждения (только при `Принята`): `Ожидает подтверждения` или `Подтверждено`.
  - Действия:
    - `Подробнее` (детальная карточка с блоками `История` и `Инструкции`).
    - `Открыть рабочее пространство` (только при `Подтверждено`, новая вкладка Bitrix24) + быстрые ссылки `К задачам`/`К документам` (если доступны).
    - `Отозвать заявку` (только для `Отправленные`, если статус `На рассмотрении`).

### 6.4 Вкладка «Уведомления»

- Хедер вкладки: заголовок
- Фильтры: чипсы `Идеи`, `Команда`, `Bitrix`, `Системные`.
- Карточка уведомления (одна строка):
  - Иконка события + зеленый бейдж `Новое` (бэйдж пропадает после первого открытия страницы с новыми уведомлениями).
  - Блок текста: заголовок (`Модератор вернул идею «Название»`) и короткое описание/причину.
  - Метаданные справа: только время и (при необходимости) бэйдж дедлайна (`Дата`, `Просрочено`).
  - Действия: кнопка `Перейти` (открывает соответствующую идею/запрос или задачу Bitrix24).

- Лэйзи-загрузка: при прокрутке вниз подгружаются новые уведомления без кнопок управления.
- Примеры (участник): «Модератор вернул идею „AI-диспетчер“ — требуется доработка», «Стейкхолдер одобрил идею „Цифровой наставник“», «Просрочена задача Bitrix по идее „AR-инструктаж“ — срок 12.07.2025».
- Примеры (стейкхолдер): «Новая идея по запросу „Автономный контроль утечек“», «Освободилась обязательная роль в команде идеи „AI-диспетчер“», «Добавлен новый документ в идее „Цифровой наставник“».

### 6.5 Вкладка «Профиль»

- Поля редактирования: аватар (кнопка `Загрузить`), ФИО, организация, должность, контакты (телефон, e-mail), интересы/роли, настройка уведомлений (чекбоксы `E-mail`, `Push`).
- Кнопка `Сохранить`, линк `Сменить пароль`.
- Кнопка `Открыть задачи в Bitrix24` (доступ появляется после участия в одобренной идее).

## 7. Личный кабинет стейкхолдера

### 7.1 Общая навигация

- Боковое меню/вкладки: `Мои запросы`, `Портфель идей`, `Уведомления`, `Админ. панель`, `Профиль`. На пункте «Мои запросы» отображается бейдж `+N`, если появились новые идеи.

### 7.2 Вкладка «Мои запросы»

- Над таблицей расположен блок числовых индикаторов `Активность по моим запросам`:
  - `Новых идей (за 7 дней)`,
  - `Активных авторов (за 7 дней)`,
  - `Идей на решение (сейчас)`,
  - `Решений принято (за 7 дней)`.
- Табличный реестр запросов:
  - Колонки: `Запрос` (название + бейдж `+N`), `Статус` (круглый индикатор `На сборе идей`/`Закрыт`), `Направление` (цветной бейдж), `Дата обновления`, `Дедлайн` (бейдж с датой `ДД.ММ.ГГГГ` или значением `Просрочено`), `+N` (непрочитанные).
  - Действия в строке: `Просмотр` (открывает карточку запроса на витрине), `Редактировать` (сразу открывает форму редактирования), `Закрыть` (при необходимости). Кнопки в отдельной колонке; решения по идеям внутри карточки запроса/идеи.
- В раскрытом запросе:
  - Блок `Комментарии` (стейкхолдер, координатор, модератор, авторы) заменяет отдельные примечания.
  - Блок `Идеи по запросу` содержит компактные карточки идей с кнопкой `Подробнее`; решения принимаются внутри полной карточки идеи.

### 7.3 Вкладка «Портфель идей»

- Над карточками расположен блок числовых индикаторов `Прогресс и риски по одобренным идеям`:
  - `Идей в реализации`,
  - `Идей с рисками` (есть просроченные задачи),
  - `Задач на 7 дней` (по открытым задачам одобренных идей).
- Фильтры-чипсы: `Все`, `Одобрена`, `В архиве`.
- Идеи в статусах `На модерации/На доработке/На рассмотрении` обрабатываются в разделе `Мои запросы` и не входят в портфель проектов.
- Карточки идей — единый формат:
  - Верх: круглый индикатор статуса, дата обновления, бейдж направления.
  - Средний блок: заголовок (кликабелен), 1–2 строки описания.
  - Низ: автор и кнопка `Подробнее`.
- **Раскрытая страница идеи из портфеля** (переход по кнопке `Просмотр/Открыть`):
  - Шапка: название идеи, круглый индикатор статуса, бейдж направления, кнопки `Перейти к задачам` и `Перейти к документам` (в новой вкладке).
  - Блок «Описание»: краткое описание, ссылка на запрос-источник (`Открыть запрос`), дата публикации и дата обновления, бейдж `+N` (количество непрочитанных обновлений).
  - Блок «Команда»: перечень текущих ролей/участников; кнопок управления нет (назначения выполняются в Bitrix24).
  - Блок «Комментарии»: единая лента сообщений (стейкхолдер, координатор, автор, участники команды) с полем ввода.
  - Блок «Вложения»: таблица файлов (имя, тип, размер, дата), кнопка скачивания/открытия.
  - Блок «Статус и метрики»: текстовые подписи «Просрочено задач: …» (данные из Bitrix24, только просмотр).
  - Футер: быстрые ссылки `Перейти к задачам`, `Перейти к документам`, кнопка `Архивировать` (только для идеи в статусе `Одобрена` и только стейкхолдеру своего направления).
    - Нажатие `Архивировать` открывает модалку `Архивировать проект` с обязательным полем `Сопроводительное сообщение`.
    - После архивации идея получает статус `Архив`, а в карточке появляется блок `Кейс`:
      - если кейс ещё не создан — кнопка `Создать кейс` (только стейкхолдер);
      - если кейс создан — статус публикации (`Черновик`/`Опубликовано`/`Снято с публикации`) и действия `Редактировать`, `Опубликовать`, `Снять с публикации` (по правам).

### 7.4 Вкладка «Админ. панель»

- Экран с кратким дашбордом задач из Bitrix24 (просмотр без управления):
  - Заголовок `Мои задачи`, подзаголовок «Активные задачи из Bitrix24».
  - Счётчики в три карточки: `Всего задач`, `Активных`, `Просроченных` (только чтение, данные из Bitrix API).
  - Кнопка `Открыть в Bitrix24` (основная) открывает административную доску/канбан стейкхолдера в новой вкладке.
  - Кнопка `Регламент` ведёт на страницу с описанием воркфлоу стейкхолдера на платформе (шаги, роли, точки входа в Bitrix24).
  - Табличный список задач: колонки `Задача` (название + короткое описание), `Дедлайн` (бейдж даты), `Проект` (ссылка/id), действие `Открыть` для перехода в задачу Bitrix24.
  - Любые операции (изменение статусов, комментарии, назначения) выполняются только во внешней доске Bitrix24; на витрине — просмотр и переходы.

### 7.5 Вкладка «Уведомления»

- Повторяет ленту участника, но добавлены фильтр `Админ. доска` и бейдж `Запрос`.
- Карточка: иконка, заголовок (`Новая идея по запросу`), краткий текст, бейдж запроса (содержит текстовое название запроса), время.
- Действия: `Открыть`, `Перейти к административной задаче` (для триггеров доски), меню `…`.
- Бейдж `+N` на карточках «Новая идея» и индикатор дедлайна для событий `Просрочена задача Bitrix`.

### 7.6 Вкладка «Профиль»

- Поля редактирования: аватар (`Загрузить`), ФИО, подразделение, контакты, роли/интересы, настройки уведомлений (чекбоксы `E-mail`, `Push`).
- Кнопки `Сохранить`, `Сменить пароль`, `Открыть задачи в Bitrix24`.

## 14. Новые страницы для Ф3 (портфель проектов и аналитика)

### 14.1 Портфель проектов (публичная витрина)

- Публичная страница, доступна без авторизации, режим «только просмотр».
- Структура: заголовок `Портфель проектов` + описание «одобренные идеи и опубликованные кейсы».
- Табы: `Одобренные идеи` / `Кейсы` (таб — единственный способ разделения; отдельный фильтр по статусу не используется).
- Фильтры (общие для обеих вкладок):
  - `Поиск по названию`.
  - `Направление` (select: `Все` + 6 направлений).
  - `Сортировка`: `По дате обновления` / `По дате одобрения или публикации` / `По направлению`.
  - `Сбросить фильтры`.

**Вкладка `Одобренные идеи`**

- Показывает только идеи со статусом `Одобрена`.
- Карточка в списке: бейдж направления, дата `Одобрена ДД.ММ`, бейдж типа `Идея`, статус‑индикатор `Одобрена`, кнопка `Открыть`.
- Страница проекта (одобренная идея) — read-only: шапка (название, направление, статус, даты), блоки `Описание`, `Материалы` (публичные ссылки), `Команда` (текущий состав без действий). Комментарии/решения/управление командой на публичной странице не показываются.

**Вкладка `Кейсы`**

- Показывает только кейсы со статусом публикации `Опубликовано` (архивные идеи без опубликованного кейса гостю не показываются).
- Карточка в списке: бейдж направления, дата `Опубликовано ДД.ММ`, бейдж типа `Кейс`, статус‑индикатор `Опубликовано`, кнопка `Открыть`.
- Страница кейса (read-only): шапка (название, направление, статус публикации, даты `Опубликовано/Обновлено`), блоки «Описание», «Кратко о результате», «Материалы» (публичные ссылки/кнопки), «Команда» (роль + инициалы/маска для гостя; для авторизованных — ФИО + роль). Никаких кнопок управления командой или публикацией на публичной странице.

### 14.3 События (календарь/список)

- Верхний фильтр: `Когда` (Будущие/Прошедшие), `Формат` (Онлайн/Офлайн), `Направление`, поиск по названию.
- Представления: календарь (месяц/неделя) и список (карточки).
- Карточка события: дата и время (начало–окончание), формат, место/ссылка, спикеры, статусная плашка `Завершено` или `Отменено`, кнопки `Зарегистрироваться` (если регистрация открыта и событие будущее), `Добавить в календарь (.ics)`, после завершения — `Материалы` (если добавлены).
- Страница события: подробное описание, программа, спикеры, блок «Материалы» (появляется после завершения), форма регистрации (ФИО, e-mail, роль, чекбокс согласия на обработку ПДн). После отправки — экран подтверждения + уведомление на e-mail.

### 14.4 ЛК: вкладка «События» (участник/стейкхолдер)

- Раздел «Мои регистрации»: таблица событий (Дата/Название/Статус/Формат/Кнопки `Перейти` или `Материалы`, `Отменить регистрацию` для будущих событий).
- Статусы регистрации: `Активна`, `Отменена пользователем`, `Отменена организатором`.
- Уведомления о событиях отображаются тем же универсальным компонентом (кнопка `Перейти`).

### 14.5 Аналитика (дашборды read-only)

- Экран `Админ/Аналитика → Сводка` (руководство): минимальный набор критичных метрик, только просмотр.
- Состав блоков:
  - `Сейчас`: `Активных запросов`, `Идей на решение`, `Проектов в реализации`, `Проектов с риском`.
  - `За период`: `Новых запросов`, `Подано идей`, `Решений принято`, `Доля одобрения`, `Архивировано проектов`, `Опубликовано кейсов`.
  - `Воронка результатов`: `Подано идей` → `Решений принято` → `Одобрено` → `Архивировано` → `Опубликовано кейсом` (с процентами конверсии).
- Фильтры сверху: `Период`, `Направление`, `Стейкхолдер`.
- Кнопка выгрузки: `Экспорт Excel` (применяет текущие фильтры). CSV/PDF не используются.
