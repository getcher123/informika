# F3 — чеклист макетов и правок

Ниже — список недостающих макетов и правок существующих макетов по требованиям Ф3. Формат: чек‑лист для контроля готовности.

## Правила ведения чек‑листа

- Отмечайте пункты по факту выполнения, не заранее.
- Чек‑лист заполняется по мере прогресса и обновляется после каждого цикла «правки → рендеры → правки».
- Для каждого макета обязателен линейный пайплайн (раздел 1).

## 1. Линейный пайплайн для каждого макета (создание/модификация)

1. [ ] Перечитать документы из раздела 2 (а для новых функций — при необходимости шаблон требований).
2. [ ] Сформировать список блоков и данных по `docs/tech-spec-F3.md`.
3. [ ] Создать/изменить макет по `Layouts/Mockups/docs/new-mockup-guide.md`.
4. [ ] Запустить рендеры 320/768/1024/1440 (`scripts/render-mockups.js`).
5. [ ] Зафиксировать замечания по рендерам и внести правки.
6. [ ] Пересвериться с `Layouts/Mockups/UNIFICATION_GUIDE.md` и `Layouts/Mockups/ui-kit.html`, привести элементы к унификации.
7. [ ] Повторить рендеры и подтвердить исправления.
8. [ ] Обновить статусы пунктов в разделах 3–4.

## 2. Документы для работы (справочник)

- `Layouts/Mockups/docs/new-mockup-guide.md` — процесс создания и чек‑лист качества.
- `Layouts/Mockups/UNIFICATION_GUIDE.md` — унификация компонентов и каркасов.
- `Layouts/Mockups/ui-kit.html` — визуальные примеры и классы.
- `docs/tech-spec-F3.md` — основной источник по наполнению макетов.
- `Layouts/Templates/bitrix/requirements-template.md` — опционально для новых функций/интеграций.

## 3. Недостающие макеты (новые страницы/экраны)

Каждый пункт проходит линейный пайплайн из раздела 1.

- [ ] Публичная страница проекта (одобренная идея, read‑only) для портфеля проектов.
- [ ] Публичная страница кейса (read‑only) с блоками «Описание / Кратко о результате / Материалы / Команда».
- [ ] Форма «Создать кейс» для стейкхолдера с полями из Ф3 (§5.1.1) и статусами публикации `Черновик → Опубликовано → Снято`.
- [ ] Публичный календарь событий (вид «календарь» + «список», фильтры `Когда/Формат/Направление/Поиск`).
- [ ] Страница события с регистрацией (форма + подтверждение).
- [ ] ЛК «События» (участник/стейкхолдер): список регистраций, статусы, действия.

## 4. Правки существующих макетов (по Ф3)

Каждый пункт проходит линейный пайплайн из раздела 1.

- [ ] `Layouts/Mockups/idea-detail.html`: добавить модалку «Архивировать проект» с обязательным полем «Сопроводительное сообщение» (стейкхолдер).
- [ ] `Layouts/Mockups/idea-detail.html` или `Layouts/Mockups/stakeholder-portfolio.html`: блок «Кейс» после архивации (статус публикации + действия `Создать/Редактировать/Опубликовать/Снять с публикации`).
- [ ] `Layouts/Mockups/portfolio.html`: убедиться, что ссылки из карточек ведут на новые публичные страницы проекта/кейса (см. раздел 1).
