# Краткое техническое задание

## 1. Назначение и цели

Платформа поддерживает инновационный цикл: стейкхолдер публикует запрос, участники подают идеи, модераторы проводят премодерацию, стейкхолдер принимает решение. После завершения работ и перевода идеи в статус «Архив» она считается переданной в проект и далее сопровождается внешними процедурами (ПК/НТК) вне платформы.

## 2. Роли и доступы

| Роль | Права в Ф1 | Права в Ф2 |
|------|------------|------------|
| Стейкхолдер | Создаёт/публикует запросы, принимает решения (CRM Bitrix). | Получает кастомный ЛК для контроля одобренных идей, отчётность, доступ к Bitrix24 задачам. |
| Участник | Подаёт идеи, управляет черновиками, отслеживает статусы в ЛК. | Работает с задачами/документами в Bitrix24 после одобрения идеи. |
| Модератор | Проводит премодерацию запросов и идей. | Запускает структуру сопровождения, следит за корректностью данных. |
| Координатор | — | Управляет правами доступа, формирует сводные отчёты, сопровождает ПК. |
| Эксперты/Менторы/Трекеры | — | Подключаются к задачам в Bitrix24 как исполнители/наблюдатели (по проектным решениям). |
| Гость | Просматривает публичную витрину. | Без изменений. |

## 3. Жизненный цикл идеи

Статусы: Черновик → На модерации → На доработке → На рассмотрении → Одобрена → Архив (→ проект вне платформы) / Отклонена.

## 4. Функциональность Фазы 1
- Банк инноваций (каталог запросов и идей, карточки, фильтры, поиск).
- Личный кабинет участника (идеи, заявки в команду, профиль).
- CRM-интерфейсы модератора и стейкхолдера (премодерация, решения).
- Базовые уведомления (модерация, решения, возврат на доработку).

## 5. Функциональность Фазы 2
- Кастомный ЛК стейкхолдера: портфель одобренных идей, отчёты, ссылки на Bitrix24.
- Интеграция с сервисом совместной работы Bitrix24: задачи, документы, уведомления, отчётность. Воркфлоу задач: Новая → В работе → На проверке → Возврат → Готово; шаблоны задач подгружаются при переходе идеи в «Одобрена». 
- Шаблоны задач и структура хранения документов для этапа «Одобрена».
- Расширенные уведомления и отчёты для координатора и стейкхолдера.
- Подготовка и подтверждение готовности к ПК/НТК реализуются только в задачах Bitrix24; витрина не содержит отдельных стадий или бейджей по этим процессам.

## 6. Навигация

Хедер (общий для всех фаз): «Банк инноваций», «Конкурсы», ссылки-заглушки «Медиасентр» и «Обучение», кнопка «Войти» (аватар при авторизации). Не реализованные разделы открываются заглушкой «Раздел появится в следующей фазе».

## 7. Ограничения
- Фаза 1: без командной работы, только идеи.
- Фаза 2: без публичных страниц проектов и внешнего BI.
- Фаза 3: события, расширенная аналитика, публичные страницы — в будущем.

## 8. Дорожная карта
- **Ф1** — реализовано: Банк инноваций, ЛК участника, CRM-интерфейсы, уведомления.
- **Ф2** — в разработке: сопровождение одобренных идей, интеграция с Bitrix24, отчётность стейкхолдера.
- **Ф3** — план: события, публичные страницы, расширенная аналитика, интеграции.

## 9. Глоссарий

| Термин | Определение |
|--------|-------------|
| Запрос | Потребность стейкхолдера, запускает сбор идей. |
| Идея | Ответ участника, пока находится в статусах до «Архива». |
| Одобрена | Статус, с которого начинается работа в Bitrix24 (задачи, документы). |
| Архив | Финальное состояние идеи внутри платформы; далее идея считается проектом (вне системы). |
| Банк инноваций | Каталог запросов и идей. |
| ЛК участника | Кастомный интерфейс для управления идеями и заявками. |
| ЛК стейкхолдера | Интерфейс Ф2 для контроля одобренных идей и отчётности. |
| Сервис совместной работы | Модуль Bitrix24 (Задачи + Диск), используемый после одобрения идеи. |
| ПК/НТК | Процедуры оценки за пределами платформы. |
