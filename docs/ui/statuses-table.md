# Статусы ключевых сущностей

| Сущность | Статусы (порядок/развилки) | Комментарии |
|----------|----------------------------|-------------|
| Запрос на инновацию | Черновик → На сборе идей → Закрыт | Черновик — подготовка и редактирование стейкхолдером; На сборе идей — опубликован, доступен для подачи идей; Закрыт — выставляется по дедлайну или вручную, скрывает кнопку подачи (источник: docs/specs/phases/tech-spec-F1.md, §4.1). |
| Идея | Черновик → На модерации → На доработке ↺ → На рассмотрении → (Одобрена → Архив) \| Отклонена | Черновик — личный черновик автора; На модерации — проверка координатором; На доработке — возврат с замечаниями (повторная отправка ограничена политикой до 3 попыток); На рассмотрении — решение стейкхолдера; Одобрена — запускает рабочее пространство Bitrix24; Архив — фиксация итога и передача во внешний проект; Отклонена — финальное решение с обоснованием. Подготовка и подтверждение готовности к ПК/НТК ведутся в Bitrix24 без отдельных статусов или бейджей на витрине (источники: docs/specs/overview.md, §3; docs/specs/phases/tech-spec-F1.md, §4.2). |
| Кейс | Черновик → На модерации → Кейс → Архив | Публичная витрина результата по завершённой идее/проекту. Черновик — подготовка публичного описания и материалов; На модерации — проверка контента (координатор/стейкхолдер), обязательная валидация материалов; Кейс — опубликован (read-only) в публичном каталоге; Архив — снят с публикации/устарел, хранится для истории (источники: docs/specs/phases/tech-spec-F3.md, §4–§7; docs/ui/interface-elements.md, §14). |
| Заявка в команду | На рассмотрении → (Ожидает E-mail подтверждения → В команде \| Отклонена) → Архив | На рассмотрении — ожидается решение модератора/стейкхолдера; Ожидает E-mail подтверждения — приглашение Bitrix24 отправлено, доступ ещё не выдан; В команде — приглашение подтверждено, доступ к рабочему пространству выдан; Отклонена — отказ (с причиной); Архив — завершённые/отозванные/истекшие записи (источники: docs/processes/automation-table.md). |
| Задача Bitrix24 (воркфлоу сопровождения) | Новая → В работе → На проверке → (Возврат ↺ \| Готово) | Воркфлоу применяется к задачам, созданным после статуса идеи «Одобрена»; Возврат переводит задачу обратно «В работе» с комментариями; флаг «Просрочено» используется как дополнительный индикатор, но не меняет статус (источник: docs/specs/overview.md, §5). |
