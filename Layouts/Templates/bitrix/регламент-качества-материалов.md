# Регламент качества материалов

## 1. Общие принципы
- Используем единый стиль Markdown / Google Docs / Office (шрифты: Inter/Arial 12 pt, межстрочный 1.15).
- Все файлы именуются `<дата>-<краткое-название>-v<версия>.md`.
- Храним документы только в папках Bitrix24 (без локальных копий в мессенджерах).

## 2. Версионирование
1. Каждое важное изменение фиксируем в разделе `История версий` внутри документа.
2. Для файлов, требующих согласования, используем схему `Черновик → На ревью → Утверждено` с указанием даты.
3. При общих правках — комментарии и предложения (`режим предложений`).

## 3. Требования к разделам
- **Отчёты / аналитика**: Введение, Методы, Результаты, Выводы, Рекомендации.
- **Тестирование**: Цель, Сценарии, Метрики, Замечания, Следующие шаги.
- **ПК/НТК пакет**: Титульный лист, Краткое описание решения, KPI, Команда, План внедрения.

## 4. Файлы и мультимедиа
- Скриншоты: PNG/JPEG ≤ 2 МБ, с подписями «Рис. 1 — ...».
- Таблицы: используем Markdown или Google Sheets (ссылка, права только просмотр/комментарии).
- Видео / демо: грузим в корпоративное хранилище, прикладываем ссылку.

## 5. Процесс ревью
- Автор публикует черновик → назначает ревьюеров (тимлид + координатор).
- Ревью проводится за 2 рабочие дня, комментарии фиксируются в документе.
- После согласования файл переносится в соответствующую папку (`/ПК-НТК`, `/Отчёты`).

## 6. Контроль качества
- Раз в неделю координатор делает выборочную проверку: соблюдение структуры, названия, наличие версий.
- Нарушения фиксируются в задаче «долг по качеству» и устраняются в течение 3 дней.
