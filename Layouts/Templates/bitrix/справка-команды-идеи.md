# Справка по команде идеи

## 1. Добро пожаловать
- Описание цели идеи / проекта.
- Основные контактные лица: стейкхолдер, координатор, тимлид.
- Краткий план этапов: Одобрена → Готово к ПК → Готово к НТК → Архив.

## 2. Каналы связи
| Канал | Назначение | Ответственный |
|-------|------------|---------------|
| Чат Bitrix24 | Ежедневные вопросы, синхронизация | Тимлид |
| Общий e-mail | Эскалации, официальные письма | Координатор |
| Созвоны (планёрка) | Планирование недели | Тимлид / Координатор |

## 3. Структура Bitrix24
- Канбан задач: ссылка `[Перейти к задачам](<вставьте ссылку>)`.
- Диск идеи: обзор папок `/Требования`, `/План`, `/Прототипы`, `/Тесты`, `/ПК-НТК`, `/Отчёты`, `/Коммуникации`.
- Обязательные документы: ссылка на актуальные шаблоны (см. раздел 5).

## 4. Роли и ответственность
| Роль | ФИО | Ответственность | Каналы |
|------|-----|----------------|--------|
| Тимлид | | План, синхронизация задач | Чат, звонки |
| Аналитик | | Требования, исследования | Канбан, Диск `/Требования` |
| Разработчик | | Реализация решения | Задачи `Прототип` |
| Тестировщик | | Сценарии тестов, отчёты | Диск `/Тесты` |
| Эксперт/Ментор | | Обратная связь, советы | Комментарии |

## 5. Обязательные документы и чек-листы
- `шаблон-требований.md` — описывает бизнес-проблему.
- `шаблон-плана-реализации.md` — дорожная карта.
- `шаблон-отчета-по-тестированию.md` — результаты испытаний.
- `чеклист-подготовки-к-пк-нтк.md` — к подготовке ПК/НТК.
- `шаблон-итогового-отчета.md` — итог проекта.

## 6. Контрольные точки
| Этап | Дата | Описание | Ответственный |
|------|------|----------|---------------|
| Старт рабочей области | | Настройка Bitrix24 | Координатор |
| Готово к ПК | | Собран пакет, чек-лист закрыт | Тимлид / Стейкхолдер |
| Готово к НТК | | Материалы согласованы | Стейкхолдер |
| Архив | | Передача результатов | Координатор |

## 7. Правила работы
- Все задачи фиксируются в Bitrix24, переписка — в комментариях задач или чате.
- Новые участники проходят `чеклист-онбординга-команды.md` в течение 2 рабочих дней.
- При любых блокерах оформляется задача `Эскалация` и уведомляется координатор.
