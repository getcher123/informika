# Task guide: правила работы с задачами

## 1. Создание задач
- Один таск = конкретный результат (`Definition of Done`).
- Описание: цель, ожидаемый результат, критерии приёмки.
- Добавляйте чек-лист, теги, дедлайн, исполнителя и наблюдателей.

## 2. Колонки канбана
- `Новая` — только что созданные.
- `В работе` — активное исполнение.
- `На проверке` — ждёт ревью или приёмку.
- `Возврат` — требуется доработка; обязательно укажите причину.
- `Готово` — завершённые задачи (с подтверждением координатора/стейкхолдера).

## 3. Комментарии и файлы
- Обновления публикуем в комментариях (не в сторонних чатах).
- Прикладываем результаты в задаче или ссылку на документ.
- Используем @упоминания для ответственных.

## 4. SLA по задачам
- Ответ на комментарий — до конца рабочего дня.
- Перевод между колонками сопровождается комментарием/описанием, что сделано.
- Просроченные задачи подсвечиваются, координатор получает уведомление.

## 5. Репорты
- Каждую пятницу исполнитель обновляет статус и время.
- Тимлид сверяет фактический прогресс с планом (`plan-template.md`).

## 6. Чек-лист закрытия
- [ ] Выполнены все пункты чек-листа.
- [ ] Результаты приложены.
- [ ] Комментарий «Готово» + дата.
- [ ] Исполнитель перевёл задачу в `На проверке` / `Готово`.
