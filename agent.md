# agent.md

В этом репозитории всю техническую информацию по 1C‑Bitrix (классы/методы/события/D7/REST/примеры) нужно находить через локальные гайды и поиск по документации, в первую очередь по инструкции:

- `docs/bitrix-docs-new/docs/AGENT.md` — единый гайд «как искать справку 1C‑Bitrix в этом Markdown‑vault» (точка входа и алгоритм поиска).

Если ответ не находится локально — сначала перепроверьте по правилам из `docs/bitrix-docs-new/docs/AGENT.md` (варианты имён, `index.md` vs `index.php.md`, регистр, транслитерация), и только потом переходите к внешним источникам.

## Правила проведения исследовательских задач (READ ONLY)
- Использовать чеклист исследования (например, `docs/deep-research-plan.md`) в **любом порядке** пунктов.
- Каждый шаг выполнять строго в режиме чтения (READ ONLY); не модифицировать файлы/БД/настройки на сервере.
- После выполнения каждого пункта добавлять короткий отчёт в отчётный документ (например, `docs/deep-research-report.md`): что сделано, что найдено, ссылки/пути, вывод.
- Если на шаге есть блокер — сначала попытаться устранить своими силами (поиск в коде, локальной документации, повторная проверка путей). Если не вышло, пропустить шаг и зафиксировать блокер в финальном саммари.
- Продолжать исследование, пока не останется пунктов без блокеров; в конце вывести список блокеров/пропущенных шагов отдельным блоком саммари.
